# 纯函数-柯里化-组合函数

# JavaScript-纯函数

# 什么是纯函数？

定义：一个函数的 输出 (返回结果)只取决于它的 输入 (参数),并且不会有修改全局变量/引用变量等的 副作用 ,这样一个函数我们便称之为纯函数。

简单来说，

  • 确定的输入产生确定的输出；
  • 函数在执行过程中不会产生副作用；

示例一：

// JavaScript
var a = 3;
var foo = (b) => a + b;
console.log(foo(2));// 5
a = 5;
console.log(foo(2));// 7

如上代码所示，示例一中的函数foo不是纯函数，因为它的 输出 并不只是取决于 输入 b,即当输入b确定时并不能确定函数foo的输出，毕竟外部变量a可能会发生变化，如示例一中代码所示，当外部变量a重新赋值后，foo(2)的输出也随之变化，确定的输入没有产生确定的输出，所以示例一中的函数foo不是纯函数。

// JavaScript
function foo(obj,b){
    obj.name = "zzwi";
    return b + 1;
};
var obj = {
    name: "why",
    age: 18
};
var result = foo(obj,2);
console.log(obj.name);// zzwi
console.log(result);// 3

上述示例二代码中的函数foo同样也不是纯函数。因为它修改了外部传进来的对象,即产生了 副作用 。

副作用 ：一个函数执行过程中产生了外部可观察的变化，比如修改了全局变量，修改参数或者改变外部存储，那么就说这个函数是有副作用的。

纯函数在执行过程中不能产生这样的副作用

  • 副作用往往是bug的"温床"。

示例三：

数组的两个方法

  • slice：slice截取数组时不会对原数组进行任何操作，而是生成一个新的数组；
  • splice：splice截取数组，会返回一个新的数组，也会对原数组进行修改；

slice就是一个纯函数，不会修改传入的参数；

splice不是纯函数，会修改传入的参数

// JavaScript
var arr = [1, 2, 3, 4, 5, 6, 7, 8, 9];
// slice截取数组时不会对原数组进行任何操作，而是生成一个新的数组
var arr_1 = arr.slice(1,6);
console.log(arr_1);// [2, 3, 4, 5, 6]
console.log(arr);// [1, 2, 3, 4, 5, 6, 7, 8, 9]

// splice截取数组，会返回一个新的数组，也会对原数组进行修改
var arr_2 = arr.splice(1,6);
console.log(arr_2);// [2, 3, 4, 5, 6, 7]
console.log(arr);// [1, 8, 9]

# 纯函数的优点

纯函数在函数式编程中非常重要，原因：

  1. 编写完成后不需要考虑上下文环境，只需要考虑输入和输出，如果传入相同的参数，它们将始终产生相同的结果。
  2. 在需要修改纯函数时，因为没有复杂的外部依赖，你可以放心地进行修改，不必担心没注意到的副作用会诞生出什么未知的bug。
  3. 由于纯函数是自给自足的，它需要的东西在输入参数中都已经声明，所以它可以移植到任何地方。

# JavaScript-柯里化

# 什么是柯里化？

在数学和计算机科学中，柯里化是把接收多个参数的函数，变成接收一个单一参数(最初函数的第一个参数)的函数，并且不断地依次返回同样接收单一参数的函数来接收余下的参数，直到参数全部接收完毕为止，最后返回结果的新函数的技术。

举例来说，一个接收3个参数的A函数，在进行柯里化后，柯里化版本的A函数接收第一个参数并返回接收第二个参数的B函数，B函数再返回一个接收第三个参数的C函数，经过函数内部相关操作后，返回最终结果。

// JavaScript
// 原函数
function foo (num1,num2,num3) {
    console.log(num1 + num2 + num3);
};
// 柯里化版本
function bar (num1) {
    return function (num2) {
        return function (num3) {
            console.log(num1 + num2 + num3);
        }
    }
};
foo(1, 2, 3); // 6
bar(1)(2)(3); // 6

而JavaScript中的柯里化概念与数学和计算机中的柯里化的概念并不完全一样。

  • 在数学和计算机科学中的柯里化函数，一次只能传递一个参数；
  • 而JavaScript中的柯里化函数一次可以传递一个或多个参数；

也就是接收一部分参数，返回一个函数接收剩余参数，接收足够参数后，执行原函数。

// JavaScript
// 原函数
function foo (num1, num2, num3) {
    console.log(num1 + num2 + num3);
};
// 柯里化函数
function bar (num1) {
    return function (num2, num3) {
        console.log(num1 + num2 + num3);
    }
};
function baz (num1, num2) {
    return function (num3) {
        console.log(num1 + num2 + num3);
    }
};
foo(1, 2, 3);// 6
bar(1)(2, 3);// 6
baz(1, 2)(3);// 6

// JavaScript
// var sum = x => y => z => {

var sum = x => y => z => x + y + z;
console.log(sum(10)(20)(30)); // 60

综上分析，柯里化函数不仅会把问题"复杂化"(将传递多个参数的一个函数变成传递部分参数的多个函数)，还会使程序性能下降(调用函数的次数增加，致使执行栈中需要存放的函数上下文增加)，那么柯里化函数为什么在函数式编程中十分重要呢？

# 柯里化的用途

为什么需要有柯里化呢？

  • 在函数式编程中，我们其实往往希望一个函数处理的问题尽可能的单一，而不是将一大堆的处理过程交给一个函数来处理；

  • 那么我们是否就可以将每次传入的参数在单一的函数中进行处理，处理完后在下一个函数中再使用处理后的结果；

# 让函数的职责单一

如下代码所示，在柯里化函数时分别对传入的参数进行处理

// JavaScript
// 柯里化的单一职责原则
function foo (a) {
    a = a + 2
    return function (b) {
        b = b * 2
        return function (c) {
            c = c * c
            console.log(a + b + c);
        }
    }
}

foo(1)(2)(3);// 16

# 自动柯里化函数

柯里化封装实现如下：

详细解释都在代码注释中~

// JavaScript
// 自动柯里化函数
// fn是待柯里化的函数
function zyCurry (fn) {
    console.log(this);// window
    // args是柯里化函数currying不定数量的参数
    function currying (...args) {
        console.log(this);// window
        /* fn.length是原函数参数长度
         当args长度>=原函数参数长度时，不进行柯里化
         直接执行原函数 */
        if (args.length >= fn.length) {
            /* 为避免自动柯里化函数调用时改变了原函数的this指向
             导致此刻的原函数调用出现bug
             所以使用显示绑定进行this的指向校正
             这里this向上层作用域寻找，即函数zyCurry中
             zyCurry的this指向window.所以这里的this指向window */
            return fn.apply(this,args)
        } else {
            // 当args长度小于原函数参数长度时，
            // 继续返回函数接收余下参数
            return function (...params) {
                /* 进行参数args和params的重构
                 这里this向上层作用域寻找，function (...params)
                 上层的this指向window，所以这里的this指向window */
                return currying.apply(this,[...args, ...params])
            }
        }
    }
	// 返回柯里化函数
    return currying
};

// 验证一下
function foo (a, b, c) {
    a = a + 2;
    b = b * 2;
    c = c * c;
    console.log(a + b + c);
};

var bar = zyCurry(foo);
// 独立调用，所以函数zyCurry的this指向window
bar(1)(2)(3);// 16 this都如上注释一般，指向window
var obj = { name: 'zzwi' };
obj.fn = bar;
obj.fn(1)(2)(3);// 16 
/* 由对象obj对函数进行调用
   函数currying第一次调用的this指向obj,剩余两次调用this都指向window
   其余的this都如上注释一般，指向window */
// 验证成功

# JavaScript-组合函数

# 什么是组合函数？

组合函数(Compose Function)函数是在JavaScript开发过程中一种对函数的使用技巧、模式：

比如我们现在需要对某一个数据进行函数的调用，执行两个函数fn1和fn2，这两个函数是依次执行的；

那么如果每次我们都需要进行两个函数的调用，操作上就会显得重复；

那么是否可以将这两个函数组合起来，自动依次调用呢？

这个过程就是对函数的组合，我们称之为组合函数。

# 简单的组合函数示例

三言两语说不清楚，上代码

// JavaScript
function add(num1) {
    return num1 + num1
};
function square(num2) {
    return num2 ** 2
};
// 组合函数
function composeFn(x, y) {
    return function(num1) {
        // console.log(y(x(num1)));
        return y(x(num1));
    };
};
var composeA = composeFn(add, square);
console.log(composeA(2));// 16

# 可复用的组合函数

未考虑太多的边界情况(edge case)

// JavaScript
// 可复用的组合函数
// fns是需要组合的函数
function composeFn(...fns) {
    // 获得fns的长度
    var length = fns.length;
    // 简单的类型判断，不考虑太多边界情况(edge case)
    for(var i = 0; i < length; i++) {
        if(typeof(fns[i]) !== 'function') {
            throw new TypeError('Expect arguments are function');
        }
    };
    function compose(...args) {
        var index = 0;
        /* 这里传入的参数args是个数组，不做解构直接传入的话，
           只能使用apply校正this指向 */
        var result = length ? fns[index].apply(this,args) : args;
        console.log(args instanceof Array);// true
        console.log(args);// [2]
        console.log(fns[index].apply(this,args));// 4
        // 以下7行是一些无聊操作，可忽略
        console.log([args]);// [[2]]
        console.log(...args);// 2
        console.log(fns[index].call(this,...args));// 4
        /* 不知为何以下这两个操作不仅没有报错，而且还有结果
           而且还是相同的结果 22 */
        console.log(fns[index].call(this,args));// 22
        console.log(fns[index].apply(this,[args]));// 22
        while(++index < length) {
            result = fns[index].call(this,result)
        };
        return result
    };
    return compose
};

function add(num1) {
    return num1 + num1
};
function square(num2) {
    return num2 ** 2
};

var composeA = composeFn(add, square);
console.log(composeA(2));// 16
