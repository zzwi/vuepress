# 17.ES11~ES12知识点

# ES11-BigInt类型

在早期的JavaScript中，我们不能正确的表示过大的数字：

  • 大于MAX_SAFE_INTEGER的数值，表示的可能是不正确的。

ES11引入新的基本数据类型 BigInt ，用于表示大的整数

  • BigInt 的表示方法是在数值的后面加字母n
// JavaScript(node环境)
// ES11之前 JS能正确使用的最大数字 MAX_SAFE_INTEGER
const maxInt = Number.MAX_SAFE_INTEGER
console.log(maxInt) // 9007199254740991
// 超过最大数字将不保证正确性
console.log(maxInt + 1) // 9007199254740992
console.log(maxInt + 2) // 9007199254740992

/* ES11新增基本数据类型：BigInt
   可以通过BigInt类型使用更大的数字，
   在数字末尾加上字母n，
   便会将Number类型转为BigInt类型
*/
const bigInt = 900719925474099100n
console.log(bigInt) // 900719925474099100n
// 不能让Number类型和BigInt类型的数直接相加
// console.log(bigInt + 10) // 会报错
console.log(bigInt + 10n) // 900719925474099110n
const num = 100
console.log(bigInt + BigInt(num)) // 900719925474099200n

const smallNum = Number(bigInt)
console.log(smallNum) // 900719925474099100

# ES11-Nullish Coalescing Operator

ES11增加了Nullish Coalescing Operator，即空值合并运算(操作)符—— ??

它和或值运算符 || 类似，一旦前一个值为true，便不再进行后面的运算，

但和或值运算符不同的是，它能识别出空字符串 "" 和数字 0

// JavaScript(node环境)
const foo = ""

const result1 = foo || "默认值"
const result2 = foo ?? "默认值"
console.log(result1) // 默认值
console.log(result2) //  

// =============================

const bar = 0

const result3 = bar || "默认值"
const result4 = bar ?? "默认值"
console.log(result3) // 默认值
console.log(result4) // 0

# ES11-Optional Chaining

可选链(Optional Chaining)也是ES11中新增的一个特性，主要作用是让我们的代码在进行null和undefined判断时更加清晰和简洁

也让代码更加安全、严谨，不会报错

// JavaScript(node环境)
const obj = {
  friend: {
    otto: {
      name: "zzwi"
    }
  }
}

if(obj.friend && obj.friend.otto) {
  console.log(obj.friend.otto.name) // zzwi
}

// 可选链的方式
console.log(obj.friend?.otto?.name) // zzwi

# ES11-Global This

在ES11之前我们希望获取JavaScript环境的全局对象，不同的环境获取的方式是不一样的

  • 比如在浏览器中可以通过this、window来获取
  • 比如在Node中我们需要通过global来获取

在ES11中对获取全局对象进行了统一的规范：globalThis

// JavaScript(node环境)
// 1.在浏览器环境下
// console.log(window)
// console.log(this)

// 2.在node环境下
console.log(global)

// 3.ES11统一标准
console.log(globalThis)

# ES11-for...in标准化

在ES11之前，虽然很多浏览器支持for...in来遍历对象类型，但是并没有被ECMA标准化。

在ES11中，对其进行了标准化，for...in是用于遍历对象的key的

// JavaScript(node环境)
const arr = [6, 5, 4, 3, 2, 1]
for(const item in arr) {
  console.log(item)
  // 0 1 2 3 4 5
}

// =============================

const obj = {
  name: "zzwi",
  age: 21,
  target: "excellent"
}
for(const item in obj) {
  console.log(item)
  // name age target
}

# ES12-FinalizationRegistry

FinalizationRegistry 对象可以让你在对象被垃圾回收时请求一个回调。

  • FinalizationRegistry 提供了这样的一种方法：当一个在注册表中注册的对象被回收时，请求在某个时间点上调用一个清理回调。(清理回调有时被称为 finalizer)；
  • 你可以通过调用 register 方法，注册任何你想要清理回调的对象，传入该对象和所含的值；
// JavaScript(Chrome浏览器环境)
let obj = { name: "zzwi" }
const registry = new FinalizationRegistry((value) => {
  console.log("对象被销毁了",value)
  // 对象被销毁了 obj对象
})

registry.register(obj,"obj对象")
// GC是不定时回收的
obj = null

# ES12-WeakRefs

如果我们默认将一个对象赋值给另外一个引用，那么这个引用是一个强引用：

  • 如果我们希望是一个弱引用的话，可以使用WeakRef
// JavaScript(Chrome浏览器环境)
// ES12: WeakRef类
// WeakRef.prototype.deref:
// > 如果原对象没有被销毁，那么可以获取到原对象
// > 如果原对象已经被销毁，那么获取到的是undefined
const finalRegistry = new FinalizationRegistry((value) => {
  console.log("注册在finalRegistry的对象，某一个被销毁",value)
  // 注册在finalRegistry的对象，某一个被销毁 obj对象
})

let obj = { name: "zzwi" }
// 弱引用
let info = new WeakRef(obj)

finalRegistry.register(obj,"obj对象")
obj = null

setTimeout(() => {
  console.log(info.deref()?.name) // undefined
  console.log(info.deref() && info.deref().name) // undefined
}, 10000)

# ES12-logical assignment operators

逻辑或运算符 ||=

类似于 += 的语法糖

// JavaScript(Chrome浏览器环境)
// 1.逻辑或运算符
let message = 0
// ES12之前
// 如果message为false/0/"", 则调用default message
// message = message || "default message"
message ||= "default message"
console.log(message) // default message

Numeric Separator(数字分隔符_) : 100_000_000

补充：

  • 数字末尾不允许使用数字分隔符

  • 如果一个数字以 _ 为开头，JavaScript会将其视为一个变量。

  • 数字分隔符可以放在数字的小数部分 : 12.300_00

  • 数字分隔符可以放在进制文本中 :

    // JavaScript
// 二进制文本
console.log(0b11_1) // 7
// 十六进制文本
console.log(0xa_b === 0xab) // true
