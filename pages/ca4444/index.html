# 监听对象的操作

通过 Object.defineProperty 的访问器属性(存取属性操作符)—— get/set ,

可以监听对象中的属性被设置或获取的过程

如下代码所示：

示例一：监听单一属性

// JavaScript(node环境)
const obj = {
  name: "zzwi",
  age: 18
}

Object.defineProperty(obj, 'name', {
  configurable: true,
  enumerable: true,
  get: function() {
    console.log("监听到obj对象的name属性被访问了")
    return this.age
  },
  set: function(value) {
    console.log("监听到obj对象的name属性被设置值")
    this.age = value
  }
})

obj.name = "bill"
console.log(obj.name) 
/* 
  监听到obj对象的name属性被设置值
  监听到obj对象的name属性被访问了
  bill
*/

# 缺点

这样做的缺点有：

  1. Object.defineProperty设计的初衷，不是为了去监听截止一个对象中所有属性的。
    • 我们在定义某些属性的时候，初衷其实是定义普通的属性，但是后面我们强行将它变成了数据属性描述符
  2. 如果我们想监听更加丰富的操作，比如新增属性、删除属性，那么Object.defineProperty是无能为力的。
  3. 综上，存取属性描述符设计的初衷并不是为了去监听一个完整的对象。

# Proxy

在ES6中，新增了一个 Proxy类 ，这个类从名字就可以看出来，是用于帮助我们创建一个 代理 的：

  • 也就是说，如果我们希望监听一个对象的相关操作，那么我们可以先创建一个代理对象(Proxy对象)
  • 之后对该对象的所有操作，都通过代理对象来完成，代理对象可以监听我们想要对原对象进行哪些操作

我们可以将上面的案例用Proxy来实现一次：

  1. 我们需要new Proxy对象，并且传入需要侦听的对象(被代理的对象)以及一个处理对象，可以称之为handler
    • const p = new Proxy(target, handler)
  2. 我们之后的操作都是直接对Proxy的操作，而不是原有的对象，因为我们需要在handler里面进行侦听

如下代码所示：

示例一：

在代理对象上进行的修改会自动同步到原对象

// JavaScript(node环境)
const obj = {
  name: "zzwi",
  age: 18
}

// new Proxy(被代理的对象, 处理对象)
const objProxy = new Proxy(obj, {})
objProxy.name = "bill"
objProxy.age = 20

console.log("objProxy.name : ",objProxy.name)
console.log("objProxy.age : ",objProxy.age)
console.log("obj.name : ",obj.name)
console.log("obj.age : ",obj.age)
/**
  objProxy.name :  bill
  objProxy.age :  20
  obj.name :  bill
  obj.age :  20
 */

# 捕捉器

如果我们想要侦听某些具体的操作，那么就可以在handler中添加对应的捕捉器(Trap):

set和get分别对应的是函数类型：

  • set函数有四个参数：
    • target : 目标对象(侦听的对象)
    • property : 将被设置的属性key
    • value : 新属性值
    • receiver : 调用的代理对象
  • get函数有三个参数：
    • target : 目标对象(侦听的对象)
    • property : 被获取的属性key
    • receiver : 调用的代理对象
  • has函数有两个参数：
    • target : 目标对象(侦听的对象)
    • property : 被判断的属性key
  • deleteProperty函数有两个参数：
    • target : 目标对象(侦听的对象)
    • property : 待删除的属性名key

如下代码所示：

// JavaScript(node环境)
const obj = {
  name: "zzwi",
  age: 18
}

const objProxy = new Proxy(obj, {
  /* target: 被代理的对象obj
     key: 监听的属性名
     newValue: 给属性传入的新值
  */
  // 1.获取值时的捕获器
  get: function(target, key) {
    console.log(`监听到对象的${key}属性被访问了`, target)
    return target[key]
  },
  // 2.设置值时的捕获器
  set: function(target, key, newValue) {
    console.log(`监听到对象的${key}属性被设置值`, target)
    target[key] = newValue
  },
  // 3.监听in的捕获器
  has: function(target, key) {
    console.log(`监听到对象的${key}属性in操作`, target)
    return key in target
  },
  // 4.监听delete的捕获器
  deleteProperty: function(target, key) {
    console.log(`监听到对象的${key}属性delete操作`, target)
    delete target[key]
  }
})

objProxy.age = 21
console.log(objProxy.age)
// in操作 
// 判断该属性是否存在于对象中，返回Boolean
console.log("name" in objProxy)
// delete操作
delete objProxy.name
console.log(objProxy)
/**
 *  监听到对象的age属性被设置值 { name: 'zzwi', age: 18 }
    监听到对象的age属性被访问了 { name: 'zzwi', age: 21 }
    21
    监听到对象的name属性in操作 { name: 'zzwi', age: 21 }
    true
    监听到对象的name属性delete操作 { name: 'zzwi', age: 21 }
    { age: 21 }
 */

# 特殊捕捉器

下面两个是针对函数对象的特殊捕捉器

handler.apply()

  • 函数调用操作的捕捉器。

handler.construct()

  • new 操作符的捕捉器。

代码如下所示：

// JavaScript(node环境)
function foo() {}

const fooProxy = new Proxy(foo, {
  /**
   * target: 目标对象
   * argArray: 参数列表
   * newTarget: 被创建出来的代理对象
   * 这里fooProxy便是newTarget
   */
  apply: function(target, thisArg, argArray) {
    console.log(`对foo函数进行apply调用`)
    return target.apply(thisArg, argArray)
  },
  construct: function(target, argArray, newTarget) {
    console.log(`对foo函数进行new调用`)
    console.log(target)
    console.log(newTarget === fooProxy)
    return new target(...argArray)
  }
})

fooProxy.apply({}, ['abc','cba'])
new fooProxy('abc','cba')
/**
 * 对foo函数进行apply调用
   对foo函数进行new调用
   [Function: foo]
   true
 */

# Reflect

Reflect也是ES6新增的一个API，它是一个对象，字面的意思是反射。

那么这个Reflect有什么用呢？

  • 它主要提供了很多操作JavaScript对象的方法，有点像Object中操作对象的方法
  • 比如Reflect.getPrototypeOf(target)类似于Object.getPrototypeOf()
  • 比如Reflect.defineProperty(target,propertyKey,attributes)类似于Object.defineProperty()

但我们已经有Object可以做这些操作，那么为什么还需要有Reflect这样的新增对象呢？

  • 这是因为在早期的ECMA规范中没有考虑到这种对 对象本身 的操作如何设计会更加规范，所以将这些API放到了Object上面
  • 但是Object作为一个构造函数，这些操作放到它身上并不合适
  • 另外还包含一些类似于 in 、delete操作符，让JS看起来是会有一些奇怪的
  • 所以在ES6中新增了Reflect，让我们把这些操作都集中到了Reflect对象上

比较Reflect和Object方法 (opens new window)

# Reflect的常见方法

Reflect中有哪些常见的方法呢？

它和Proxy是一一对应的，也是13个：

Reflect.getPrototypeOf(target)

  • 类似于 Object.getPrototypeOf()

Reflect.setPrototypeOf(target, prototype)

  • 设置对象原型的函数-返回一个 Boolean，如果更新成功，则返回true

Reflect.isExtensible(target)

  • 类似于 Object.isExtensible()

Reflect.preventExtensions(target)

  • 类似于 Object.preventExtensions().返回一个 Boolean

Reflect.getOwnPropertyDescriptor(target, propertyKey)

  • 类似于 Object.getOwnPropertyDescriptor().如果对象中存在该属性，则返回对应的属性描述符，否则返回 undefined

Reflect.defineProperty(target, propertyKey, attributes)

  • 和 Object.defineProperty() 类似，如果设置成功就会返回true

Reflect.ownKeys(target)

  • 返回一个包含所有自身属性(不包含继承属性)的数组。(类似于Object.keys(),但不会受enumerable影响)

Reflect.has(target, propertyKey)

  • 判断一个对象是否存在某个属性，和 in 运算符的功能完全相同

Reflect.get(target, propertyKey[, receiver])

  • 获取对象身上某个属性的值，类似于target[name]

Reflect.set(target, propertyKey, value[, receiver])

  • 将值分配给属性的函数。返回一个Boolean，如果更新成功，则返回true

Reflect.deleteProperty(target, propertyKey)

  • 作为函数的delete操作符，相当于执行 delete target[name]

Reflect.apply(target, thisArgument, argumentsList)

  • 对一个函数进行调用操作，同时可以传入一个数组作为调用参数。和Function.prototype.apply()功能类似

Reflect.construct(target, argumentsList[, newTarget])

  • 对构造函数进行 new 操作，相当于执行 new target(...args)

# Reflect的使用

将之前Proxy案例修改为Reflect来操作

如下代码所示：

// JavaScript(node环境)
const obj = {
  name: "zzwi",
  age: 18
}

const objProxy = new Proxy(obj, {
  get: function(target, key, receiver) {
    console.log("get~~~~~~~~~~~")
    // return target[key]
    return Reflect.get(target, key)
  },
  set: function(target, key, newValue, receiver) {
    console.log("set~~~~~~~~~~~")
    // target[key] = newValue
    // Reflect.set(target, key, newValue)
    const result = Reflect.set(target, key, newValue)
    // Reflect.set()返回一个Boolean值
    if(result) {} else {}
  }
})

objProxy.name = "bill"
console.log(objProxy.name)
/**
  set~~~~~~~~~~~
  get~~~~~~~~~~~
  bill
 */

# Receiver的作用

我们发现在使用getter、setter的时候有一个receiver的参数，它的作用是什么？

  • 如果我们的源对象(obj)有setter、getter的访问器属性，那么可以通过receiver来改变里面的this指向
  • 只有setter、getter这两个访问器属性拥有receiver参数

如下代码所示：

// JavaScript(Chrome浏览器环境)
const obj = {
  _name: "zzwi",
  get name() {
    return this._name
  },
  set name(newValue) {
    this._name = newValue
  }
}

const objProxy = new Proxy(obj, {
  /**
   * target是被代理对象，也就是obj
   * key是被监听的属性名
   * receiver是创建出来的代理对象，也就是objProxy
   * get和set的最后一个参数(receiver)，
   * 可以修改源对象(obj)的this指向，
   * 真正实现代理对象的中间层代理，避免直接操作源对象
   */
  get: function(target, key, receiver) {
    console.log("get---------", key)
    console.log(receiver === objProxy)
    return Reflect.get(target, key, receiver)
  },
  set: function(target, key, newValue, receiver) {
    console.log("set---------", key)
    Reflect.set(target, key, newValue, receiver)
  }
})

/**
 * 使用receiver参数修改源对象this指向后，
 * 访问或设置属性/方法将会分别调用两次get或set
 */
console.log(objProxy.name)
console.log(objProxy)
/**
  get--------- name
  true
  get--------- _name
  true
  zzwi
  Proxy {_name: 'zzwi'}
 */

objProxy.name = "bill"
/**
  set--------- name
  set--------- _name
 */

# Reflect.construct

当我们有这样一个需求：

创建一个内容是构造函数A的函数体但类型是构造函数B的新对象

这时候使用Reflect.construct就可以轻松地完成该需求

如下代码所示：

// JavaScript(node环境)
function Student(name, age) {
  this.name = name
  this.age = age
}

function Teacher() {}

// Reflect.construct(提供函数体的构造函数, 参数数组, 目标类型)
const teacher = Reflect.construct(Student, ["zzwi", 18], Teacher)
console.log(teacher) // Teacher { name: 'zzwi', age: 18 }
console.log(teacher.__proto__ === Teacher.prototype) // true

# 什么是响应式？

我们先来看一下响应式意味着什么？我们来看一段代码：

  • m有一个初始化的值，有一段代码使用了这个值
  • 那么在m有一个新的值时，这段代码可以自动重新执行
// JavaScript
let m = 20
console.log(m)
console.log(m * 2)
m = 40

# 响应式函数设计

首先，执行的代码中可能不止一行代码，所以我们可以将这些代码放到一个函数中：

  • 那么我们的问题就变成了，当数据发生变化时，自动去执行某一个函数

但是有一个问题：在开发中我们是有很多的函数的，我们如何区分一个函数需要响应式，还是不需要响应式呢？

  • 很明显，下面的函数中foo需要在obj的name发生变化时，重新执行，做出响应
  • bar函数是一个完全独立于obj的函数，它不需要执行任何响应式的操作
// JavaScript
function foo() {
    let newName = obj.name
    console.log(obj.name)
}

function bar() {
    const result = 20 + 30
    console.log(result)
    console.log("Hello World")
}

# 响应式函数的实现watchFn

续上节的问题，我们怎么区分呢？

  • 这个时候我们封装一个新的函数watchFn
  • 凡事传入到watchFn的函数，就是需要响应式的
  • 其他默认定义的函数都是不需要响应式的

如下代码所示：

// JavaScript(node环境)
const reactiveFns = []
// 响应式函数封装
function watchFn(fn) {
  reactiveFns.push(fn)
  fn()
}

const obj = {
  name: "zzwi",
  age: 18
}

watchFn(function() {
  let newName = obj.name
  console.log(obj.name)
})

watchFn(function() {
  console.log("my name is " + obj.name)
})

function bar() {
  console.log("这是个不需要响应式的普通函数")
}

// 响应
obj.name = "bill"
reactiveFns.forEach(fn => {
  fn()
})

/**
  zzwi
  my name is zzwi
  bill
  my name is bill
 */

# 响应式依赖的收集

目前我们收集的依赖是放到一个数组中来保存的，但是这里会存在数据管理的问题：

  • 我们在实际开发中需要监听很多对象的响应式
  • 这些对象需要监听的不只是一个属性，它们很多属性的变化，都会有对应的响应式函数
  • 我们不可能在全局维护一大堆的数组来保存这些响应式函数

所以我们要设计一个类，这个类用于管理某一个对象的某一个属性的所有响应式函数：

  • 相当于替代了原来的简单 reactiveFns 的数组

如下代码所示：

// JavaScript(node环境)
// 依赖收集类的封装
class Depend {
  constructor() {
    this.reactiveFns = []
  }
  addDepend(fn) {
    this.reactiveFns.push(fn)
  }
  notify() {
    this.reactiveFns.forEach(fn => {
      fn()
    })
  }
}

const dep = new Depend()
function watchFn(fn) {
  dep.addDepend(fn)
  fn()
}

const obj = { name: "zzwi", age: 18 }
watchFn(function() {
  console.log(obj.name)
})
watchFn(function() {
  console.log(obj.age)
})

obj.name = "bill"
obj.age = 20
dep.notify()
/**
  zzwi
  18
  bill
  20
 */

# 监听对象的变化

那么我们接下来就可以通过之前学习的方式来监听对象的变量：

  • 方式一：通过 Object.defineProperty 的方式（vue2采用的方式）
  • 方式二：通过 new Proxy 的方式（vue3采用的方式）

我们这里先以Proxy的方式来监听：

# 正确的依赖收集

我们之前收集依赖的地方是在 watchFn 中：

  • 但是这种收集依赖的方式我们根本不知道是哪一个key的哪一个depend需要收集依赖
  • 你只能针对一个单独的depend对象来添加你的依赖对象

那么正确的应该是在哪里收集呢？应该在我们调用了Proxy的get捕获器时

  • 因为如果一个函数中使用了某个对象的key，那么它应该被收集依赖
// JavaScript(node环境)
// 依赖收集类的封装
class Depend {
  constructor() {
    this.reactiveFns = []
  }
  addDepend(fn) {
    this.reactiveFns.push(fn)
  }
  notify() {
    this.reactiveFns.forEach(fn => {
      fn()
    })
  }
}

const depend = new Depend()
// 封装一个响应式的函数
let activeReactiveFn = null
function watchFn(fn) {
  activeReactiveFn = fn
  fn()
  activeReactiveFn = null
}

// 封装一个获取depend的函数
const targetMap = new WeakMap()
function getDepend(target, key) {
  // 根据target对象获取map的过程
  let map = targetMap.get(target)
  if(!map) {
    map = new Map()
    targetMap.set(target, map)
  }

  // 根据key获取depend对象
  let depend = map.get(key)
  if(!depend) {
    depend = new Depend()
    map.set(key, depend)
  }
  return depend
}

// 对象
const obj = { name: "zzwi", age: 18 }

// 监听对象的属性变量：Proxy
const objProxy = new Proxy(obj, {
  get: function(target, key, receiver) {
    // 根据target.key获取对应的depend
    const depend = getDepend(target, key)
    // 给depend对象中添加响应式函数
    depend.addDepend(activeReactiveFn)
    return Reflect.get(target, key, receiver)
  },
  set: function(target, key, newValue, receiver) {
    Reflect.set(target, key, newValue, receiver)
    // depend.notify()
    // 依赖收集的管理
    const depend = getDepend(target, key)
    depend.notify()
  }
})

let j = 0

watchFn(function() {
  console.log(objProxy.name,"-------",++i)
})
watchFn(function() {
  console.log(objProxy.age,"-------",++j)

# 对Depend类的重构

但是这里有两个问题：

  1. 问题一：如果函数中有用到两次key，比如name，那么这个函数会被收集两次
  2. 问题二：我们并不希望将添加reactiveFn放到get中，因为它是属于Dep的行为

所以我们需要对Depend类进行重构：

  • 解决问题一的方法：不使用数组，而是使用Set
  • 解决问题二的方法：添加一个新的方法，用于收集依赖

如下代码所示：

// JavaScript(node环境)
// 暂时存放响应式函数的对象
let activeReactiveFn = null
/**
 * Depend优化：
 *  1. depend方法
 *  2. 使用Set来保存依赖函数，而不是数组[]
 */
// 依赖收集类的封装
class Depend {
  constructor() {
    // Set 元素覆盖,自动去重
    this.reactiveFns = new Set()
  }
  // addDepend(fn) {
  //   this.reactiveFns.push(fn)
  // }
  depend() {
    if(activeReactiveFn) {
      this.reactiveFns.add(activeReactiveFn)
    }
  }
  notify() {
    this.reactiveFns.forEach(fn => {
      fn()
    })
  }
}

const depend = new Depend()
// 封装一个响应式的函数
function watchFn(fn) {
  activeReactiveFn = fn
  fn() // 自动执行一次
  activeReactiveFn = null
}

// 封装一个获取depend的函数
const targetMap = new WeakMap()
function getDepend(target, key) {
  // 根据target对象获取map的过程
  let map = targetMap.get(target)
  if(!map) {
    map = new Map()
    targetMap.set(target, map)
  }

  // 根据key获取depend对象
  let depend = map.get(key)
  if(!depend) {
    depend = new Depend()
    map.set(key, depend)
  }
  return depend
}

// 对象
const obj = { name: "zzwi", age: 18 }

// 监听对象的属性变量：Proxy
const objProxy = new Proxy(obj, {
  get: function(target, key, receiver) {
    // 根据target.key获取对应的depend
    const depend = getDepend(target, key)

    Reflect.set(target, key, newValue, receiver)

    // 依赖收集的管理
    const depend = getDepend(target, key)
    depend.notify()

})

# 对象的响应式操作-vue3

我们目前的响应式是针对于obj一个对象的，我们可以创建出来一个函数，让所有的对象都可以变成响应式对象

如下代码所示：

// JavaScript(node环境)
// 暂时存放响应式函数的对象
let activeReactiveFn = null
/**
 * Depend优化：
 *  1. depend方法
 *  2. 使用Set来保存依赖函数，而不是数组[]
 */
// 依赖收集类的封装
class Depend {
  constructor() {

    this.reactiveFns = new Set()

  //   this.reactiveFns.push(fn)
  // }
  depend() {
    if(activeReactiveFn) {
      this.reactiveFns.add(activeReactiveFn)
    }
  }
  notify() {
    this.reactiveFns.forEach(fn => {
      fn()
    })
  }
}

const depend = new Depend()
// 封装一个响应式的函数
function watchFn(fn) {
  activeReactiveFn = fn
  fn() // 自动执行一次
  activeReactiveFn = null
}

// 封装一个获取depend的函数
const targetMap = new WeakMap()
function getDepend(target, key) {

  let map = targetMap.get(target)

// 创建出一个能将对象转为响应式对象的函数---vue3
function reactive(obj) {

      // depend.addDepend(activeReactiveFn)
      depend.depend()

      // 依赖收集的管理

# 对象的响应式操作-vue2

我们前面所实现的响应式代码，其实就是Vue3中的响应式原理：

  • Vue3主要是通过Proxy来监听数据的变化以及收集相关的依赖
  • Vue2中通过我们前面学习过的 Object.defineProperty 的方式来实现对象属性的监听

我们可以将reactive函数进行如下的重构：

  • 在传入对象时，我们可以遍历所有的key，并且通过属性存储描述符来监听属性的获取和修改
  • 在setter和getter方法中的逻辑和前面的Proxy是一致的

如下代码所示：

// JavaScript(node环境)
// 暂时存放响应式函数的对象
let activeReactiveFn = null
/**
 * Depend优化：
 *  1. depend方法
 *  2. 使用Set来保存依赖函数，而不是数组[]
 */
// 依赖收集类的封装
class Depend {
  constructor() {
    // Set 元素覆盖,自动去重
    this.reactiveFns = new Set()
  }
  // addDepend(fn) {
  //   this.reactiveFns.push(fn)
  // }
  depend() {
    if(activeReactiveFn) {
      this.reactiveFns.add(activeReactiveFn)
    }
  }
  notify() {
    this.reactiveFns.forEach(fn => {
      fn()
    })
  }
}

const depend = new Depend()
// 封装一个响应式的函数
function watchFn(fn) {
  activeReactiveFn = fn
  fn() // 自动执行一次
  activeReactiveFn = null
}

// 封装一个获取depend的函数
const targetMap = new WeakMap()
function getDepend(target, key) {
  // 根据target对象获取map的过程
  let map = targetMap.get(target)
  if(!map) {
    map = new Map()
    targetMap.set(target, map)
  }

  // 根据key获取depend对象
  let depend = map.get(key)
  if(!depend) {
    depend = new Depend()
    map.set(key, depend)
  }
  return depend
}

// 创建出一个能将对象转为响应式对象的函数---vue2
function reactive(obj) {
  Object.keys(obj).forEach(key => {
    let value = obj[key]
    Object.defineProperty(obj, key, {
      get: function() {
        const depend = getDepend(obj, key)
        depend.depend()
        return value
      },
      set: function(newValue) {
        value = newValue
        const depend = getDepend(obj, key)
        depend.notify()
      },
    })
  })
  return obj
}

// 对象
const obj = { name: "zzwi", age: 18 }
const info = { address: "广州市" }

// 创建响应式对象
const objProxy = reactive(obj)
const infoProxy = reactive(info)

// 响应式函数
watchFn(function() {
  console.log(objProxy.name,"------")
})
watchFn(function() {
  console.log(infoProxy.address,"++++++")
})
objProxy.name = "bill"
infoProxy.address = "北京市"

  zzwi ------
  广州市 ++++++
  bill ------
  北京市 ++++++
