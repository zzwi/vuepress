# Vue3组件化开发(part1)

# 什么是组件

平常开发中，如果我们将所有的代码、逻辑都放在一个文件中，该文件会变得非常的臃肿和难以维护；

组件化开发的思想就是对这个庞大的文件进行拆解，抽离出一个个模块，放入不同的文件中，并进行导出，这就是组件了，易于维护且易于扩展；

需要使用时，对这些组件进行导入，组成一个完整的应用程序。

# 组件的拆分

父子组件

如下图所示：

App是Header、Main、Footer共同的父组件，

反之，Header、Main、Footer则都是App的子组件，

同理，Main与Banner、NavBar也是一样的父子组件关系。

# 组件的通信

在开发过程中，我们经常会遇到组件之间相互进行通信的需求：

  • 比如App使用了多个Header，每个地方的Header展示的内容不同，那么我们就需要传给每个Header不同的数据，让其进行展示；
  • Main同理；
  • 又比如 子组件中发生了某些事件 (点击、滚动等)，需要 父组件响应并完成某些操作 ，那就需要子组件向父组件发送事件；

总之，在一个Vue项目中，组件之间的通信是非常重要的环节，所以接下来我们就具体学习一下组件之间是如何相互之间传递数据的；

父子组件之间通信的方式

  • 父组件传递给子组件 : 通过 props 属性；
  • 子组件发送给父组件 : 通过 $emit 发送；

什么是props呢？

  • props是你可以在组件上注册一些自定义的attribute；
  • 父组件给这些attribute赋值，子组件通过attribute的名称获取到对应的值；

props有两种常见的用法：

  1. 字符串数组 ，数组中的字符串就是attribute的名称；
  2. 对象类型 ，我们可以在指定attribute名称的同时，指定它需要传递的类型、是否是必须的、默认值等等；

# props的对象用法

数组用法中我们只能说明传入的attribute的名称，并 不能对其进行任何形式的限制 ，

然而 ，当使用对象语法的时候，我们 可以对传入的内容进行限制 ：

  • 比如指定传入的 attribute的类型 ；
  • 比如指定传入的 attribute是否是必传的 ；
  • 比如指定没有传入时， attribute的默认值 ；
<!-- son.vue子组件 -->
<template>...</template>
<script>
  export default {
    props: {
      // 指定类型
      title: String,
      // 指定类型、必选、默认值
      content: {
        type: String,
        require: true,
        default: "数据"
      }
    }
  }
</script>

type类型可以是以下这些：

  • String
  • Number
  • Boolean
  • Array
  • Object
  • Date
  • Function
  • Symbol

对象类型的其他写法：

<!-- son.vue子组件 -->
<template>...</template>
<script>
  export default {
    props: {
      // 基础的类型检查(`null`和`undefined`会通过任何类型验证)
      propA: Number,
      // 多个可能的类型
      propB: [String, Number],
      // 必填的字符串
      propC: {
        type: String,
        required: true
      },
      // 带有默认值的数字
      propD: {
        type: Number,
        default: 100
      },
      // 带有默认值的对象
      propE: {
        type: Object,
        // 对象或数组的默认值必须从一个工厂函数获取
        default() {
          return { message: 'hello' }
        }
      },
      // 自定义验证函数
      propF: {
        validator(value) {
          // 这个值必须匹配下列字符串中的一个
          return ['success','warning','danger'].includes(value)
        }
      },
      // 具有默认值的函数
      propG: {
        type: Function,
        // 与对象或数组默认值不同，这不是一个工厂函数，这是一个用作默认值的函数
        default() {
          return 'Default function'
        }
      }
    }
  }
</script>

对象或数组的默认值为什么必须从一个工厂函数获取？

原因： 当被多次调用时，函数每次都会返回一个新的对象/数组，互不干扰；

反之，如果不设置工厂函数，直接return一个对象，当某个子组件对其进行修改，其他的引用也都会被修改。

prop的大小写命名

  • HTML中的attribute名是 大小写不敏感 的，所以 浏览器会把所有大写字符解释为小写字符 ；
  • 这意味着当你 使用DOM中的模板 时， camelCase(驼峰命名法) 的prop名需要使用 其等价的 kebab-case(短横线分隔命名) 命名；
  • 但是 ，在vue文件的template模板中，是可以使用 camelCase(驼峰命名法) 的，为什么？因为vue文件是由vue-loader解析的，而不是浏览器；

# 非prop的attribute

什么是非prop的attribute？

  • 当我们传递给A组件一个b属性，但是 A组件内并没有定义b属性对应的props或者emits 时，就称b属性为非prop的attribute；
  • 常见的包括class、style、id属性等；

attribute继承

  • 当 组件有单个根节点 时，非Prop的attribute将自动添加到根节点的attribute中

  <son class="target"></son>

禁用attribute继承和多根节点

如果我们 不希望组件的根元素继承attribute ，可以在组件中设置 inheritAttrs: false :

  • 禁用attribute继承的常见情况是 需要将attribute应用于根元素之外的其他元素 ；
  • 我们可以通过 $attrs 来访问所有的非prop的attribute；
<!-- son.vue子组件 -->
<template>
  <div>
    <!-- 加了v-bind语法糖:才能访问$attrs，
         否则只是添加了一个普通的class属性 -->
    <h2 :class="$attrs.class"></h2>  
  </div>
</template>
<script>
  export default {
    // 禁用attribute继承
    inheritAttrs: false
  }
</script>

# 子组件传给父组件

什么情况下子组件需要传递内容到父组件呢？

  • 当 子组件有一些事件发生时需要父组件响应 的时候，比如在子组件中发生了点击事件，父组件需要切换对应内容；
  • 子组件 有一些内容想要传递给父组件 的时候；

我们如何完成 子组件向父组件发送事件/内容 的操作呢？

  • 首先，我们需要给 子组件中的A事件触发 的情况，定义好传给父组件的 发送事件的名称B ；
  • 其次， 在父组件中用v-on监听子组件传递上来的发送事件B ，并 绑定对应的方法C ；
  • 最后，当子组件的A事件 真正触发 时，父组件中对发送事件B的 监听事件 也会被触发，然后 调用方法C ；

子组件通过$emit向父组件发送事件

  • 封装一个CounterOperation.vue的子组件
  • 内部是监听两个按钮的点击，点击之后通过 this.$emit 的方式发送事件给父组件

App.vue 是父组件， CounterOperation.vue 是子组件，

通过点击子组件，修改父组件的数字，组成一个简单的计数器 ，

代码如下：

<!-- CounterOperation.vue 子组件 -->
<template>
  <div class="CounterOperation">
    <button @click="increment">+1</button>
    <button @click="decrement">-1</button>
  </div>
</template>

<script>
export default {
  name: 'CounterOperation',
  // 注册发送事件(数组形式)
  emits: ["addOne", "subOne"],
  methods: {
    increment() {
      // 事件发送
      this.$emit("addOne");
    },
    decrement() {
      // 事件发送
      this.$emit("subOne");
    }
  }
 }
</script>

# 组件通信案例练习

想要实现的效果如下：

头部的 衣服、鞋子、裤子 这一块区域是名为 tab-control.vue 的子组件，

底部的 衣服页面、鞋子页面、裤子页面 这一块区域是父组件 App.vue 响应子组件 tab-control.vue 所显示的内容。

编写的代码如下：

<!-- tab-control.vue 子组件 -->
<template>
  <div class="tab-control">
    <div class="tab-control-item"
         v-for="(title,index) in titles"
         :key="title"
         :class="{active: currentIndex === index}"
         @click="itemClick(index)">
      <span>{{title}}</span>   
    </div>
  </div>
</template>
<script>
export default {
  props: {
    titles: {
      type: Array,
      default() {
        return []
      }
    }
  },
  data() {
    return {
      currentIndex: 0
    }
  },
  methods: {
    itemClick(index) {
      this.currentIndex = index,
      this.$emit("titleClick", index)
    }
  }
}
</script>
<style scoped>
  .tab-control{
    display: flex;
    width: 80vw;
  }
  .tab-control-item{
    flex: 1;
    text-align: center;
    cursor: pointer;
  }
  .tab-control-item.active{
    color: red;
  }
  .tab-control-item.active span{
    border-bottom: 3px solid red;
    padding: 5px 10px;
  }
</style>
