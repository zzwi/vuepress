# 认识防抖和节流函数

防抖和节流 的概念其实最早并不是出现在软件工程中，防抖是出现在电子元件中，节流出现在流体流动中

  • 而JavaScript是事件驱动的，大量的操作会触发事件，加入到事件队列中处理
  • 而对于某些频繁的事件处理会造成性能的损耗，我们就可以通过防抖和节流来限制事件频繁的发生

防抖和节流函数目前已经是前端实际开发中两个非常重要的函数，也是面试经常问到的面试题。

关于防抖和节流的问题有不少，如下：

  • 防抖和节流有什么区别？
  • 如何应用？
  • 内部原理？
  • ……

接下来让我们一起学习防抖和节流函数：

  • 区分防抖和节流
  • 知晓应用场景
  • 理解原理
  • 自己编写

# 防抖debounce函数

防抖函数到底在执行过程中做了什么？

我的理解：

  • 将触发事件对应的函数延迟执行
  • 在【延迟期】内同一事件再次触发则重新延迟
  • 比如【延迟期】是250ms，在100ms时该事件再次触发，则将事件的对应函数再次延迟250ms执行
  • 以此类推
  • 或者直接理解为 打断施法

我们拿一个例子来看看，比如：现在大部分网页、程序都带有的【返回顶部】按钮，

这个按钮会在我们向下滑动一定距离之后出现，顾名思义，点击该按钮后会让我们返回网页顶部。

那么如何让这个按钮在我们将页面滑动到一定位置时隐藏和出现呢？

监听滚动事件 。

现在进行一个简单监听，如下代码：

    var scrollTop = document.body.scrollTop || document.documentElement.scrollTop;
　　console.log('滚动条位置：' + scrollTop);
}

# 手写debounce版本1：

// debounce.js
/*
* fn [function] 需要防抖的函数
* delay [number] 毫秒，防抖期限值
*/
function debounce(fn, delay) {
  /* 1. 定义了一个名为timer的容器，
    用来保存上一次的定时器，
    同时是闭包的自由变量*/
  let timer = null
  // 2. 真正执行的函数，同时也是一个闭包
  const _debounce = function() {
    // 取消上一次的定时器
    if(timer) clearTimeout(timer)
    // 延迟执行
    timer = setTimeout(() => {
      // 需要进行防抖操作的函数
      // 这里相当于window.fn()
      // 所以this指向window
      // 后续需要改进
      fn()
    }, delay)
  }
  return _debounce
}

下面举另外一个例子：搜索框

  • 通常，搜索框都会配备联想功能，比如我输入字母a，为了更好的用户体验，一般就会出现对应的联想内容，这些联想内容通常是保存在服务器的，所以需要发送一次网络请求
  • 当我们继续输入字符，它便会继续发送网络请求
  • 如果我们输入的是alphaGo，那么一共需要发送7次网络请求
  • 这将大大损耗我们整个系统的性能，无论是前端的事件处理，还是对于服务器的压力

我们需要这么多次的网络请求吗？

  • 事实上，并不需要。正确的做法应该是在合适的情况下再发送网络请求；
  • 比如用户快速地输入了一个alphaGo，那么应当只发送一次网络请求；
  • 如果用户输入一个a后便停止(一段时间)，这个时候也应该发送一次网络请求；
  • 也就是我们应该监听用户在某个事件，比如500ms内，没有再次触发事件时，再发送网络请求

# 手写debounce版本3：

防抖函数很好用，但如果事件频繁触发的时间一直持续，那么对应的函数便会被一直延迟，当然这种情况很少见。

或许可以这样说，产品方面要求我们在第一次事件触发时就执行一次对应的函数，之后的事件触发再进行一个延迟，这样做的目的是为了更好的用户体验。

接下来就让我们来实现这个功能吧~

首先 ，在这里贴一个小知识，我们在开发过程中应该尽量避免在函数中修改从外部传进来的参数，这是我们应该遵循的代码规范，也是我们应该去培养的好习惯~

所以，不能对下面debounce函数的参数做修改。

思路：

我们可以给debounce函数上再传入一个参数immediate，传入Boolean值用来确定是否开启第一次立即执行；

可以用参数immediate做if判断是否执行timer定时器函数；

但是immediate参数的值是固定的，且在函数内应该尽量避免对其做修改 ，那么在执行判断上就有诸多不便之处，比如我们只是想立即执行第一次，但immediate作判断可能就让所有触发事件的对应函数都立即执行了；

解决这个问题，我们可以在debounce函数内部声明一个变量isInvoke，用immediate和isInvoke一起作判断；

并且，应当是我们的每一段重新输入，都能立即执行第一次，也就是在上一段防抖函数的【延时期】结束然后执行最后一次事件触发的对应函数之后，我们再重新频繁地触发事件时，第一次的事件触发仍然应该立即执行，

当然，也可以不用每段都如此，虽然这是目前的debounce主流，但我们也可以根据特殊情况进行特殊改进，比如： 只对第一段输入的第一个触发事件的对应函数进行立即执行 之类的，但若没有特殊情况，还是推荐使用主流设计；

// debounce.js
/*
* fn [function] 需要防抖的函数
* delay [number] 毫秒，防抖期限值
* immediate [boolean] 布尔值，判断是否立即执行
*/
function debounce(fn, delay, immediate=false) {
  let timer = null
  // 声明变量isInvoke
  let isInvoke = false
  const _debounce = function(...args) {
    if(timer) clearTimeout(timer)
    // 判断是否立即执行
    if(immediate && !isInvoke) {
      fn.apply(this, args)
      // 修改isInvoke为true
      isInvoke = true
    } else {
      timer = setTimeout(() => {
        fn.apply(this, args)
        // 修改isInvoke为false
        isInvoke = false
      }, delay)
    }
  }
  return _debounce
}

# 手写debounce版本4：

到了这里，我们还可以再给我们的debounce函数添加一个功能—— cancel ，这个功能是用来干嘛的呢？顾名思义，用来取消某一个事件，我们需要用它来取消还未执行的函数！

还是借用搜索框的案例，当用户在输入字符之后，网络请求发送之前，取消了这次的输入，比如点击了搜索框旁边配置的取消按钮、删除了之前所写、直接关闭该页面等等，这时候我们就可以取消对应的函数执行，拦截下这一次的网络请求，从而优化系统性能。

那么就让我们来编写我们的debounce函数版本4吧~

// ./04_debounce-v4-取消功能.js
/*
* fn [function] 需要防抖的函数
* delay [number] 毫秒，防抖期限值
* immediate [boolean] 布尔值，判断是否立即执行
*/
function debounce(fn, delay, immediate=false) {
  let timer = null
  let isInvoke = false
  const _debounce = function(...args) {
    if(timer) clearTimeout(timer)
    if(immediate && !isInvoke) {
      fn.apply(this, args)
      isInvoke = true
    } else {
      timer = setTimeout(() => {
        fn.apply(this, args)
        timer = null
        isInvoke = false
      }, delay)
    }
  }

  // 封装取消功能
  _debounce.cancel = function() {
    if(timer) clearTimeout(timer)
    timer = null
    isInvoke = false
  }
  
  return _debounce
}

实现这个功能有两种方案，

  • 一是再给debounce传入一个参数 resultCallback ，

  • 二是使用Promise。

方案一：

// ./05_debounce-v5-函数返回值.js
/*
* fn [function] 需要防抖的函数
* delay [number] 毫秒，防抖期限值
* immediate [boolean] 布尔值，判断是否立即执行
* resultCallback [function] 以fn返回值做参数的函数
*/
function debounce(fn, delay, immediate=false, resultCallback) {
  let timer = null
  let isInvoke = false
  const _debounce = function(...args) {
    if(timer) clearTimeout(timer)
    if(immediate && !isInvoke) {
      // 实际执行函数的返回值
      const result = fn.apply(this, args)
      // 将返回值抛出
      if(resultCallback) resultCallback(result)
      isInvoke = true
    } else {
      timer = setTimeout(() => {
        // 实际执行函数的返回值
        const result = fn.apply(this, args)
        // 将返回值抛出
        if(resultCallback) resultCallback(result)
        timer = null
        isInvoke = false
      }, delay)
    }
  }

  _debounce.cancel = function() {
    if(timer) clearTimeout(timer)
    timer = null
    isInvoke = false
  }

  return _debounce
}

  const _debounce = function(...args) {
    // return一个Promise
    return new Promise((resolve, reject) => {
      if(timer) clearTimeout(timer)
      if(immediate && !isInvoke) {
        // 实际执行函数的返回值
        const result = fn.apply(this, args)
        // 将返回值抛出
        resolve(result)
        isInvoke = true
      } else {
        timer = setTimeout(() => {
          // 实际执行函数的返回值
          const result = fn.apply(this, args)
          // 将返回值抛出
          resolve(result)
          timer = null
          isInvoke = false
        }, delay)
      }
    })
  }

  _debounce.cancel = function() {
    if(timer) clearTimeout(timer)
    timer = null
    isInvoke = false
  }

  return _debounce
}
