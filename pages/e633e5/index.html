Vue3-动画实现

# Vue3-动画实现

在开发中，有时候我们会想要给一个组件的 显示和消失 添加某种 过渡动画 ，这样可以在一定程度上提高用户体验，

React框架本身并没有提供任何动画相关的API，所以在React中使用过渡动画我们需要使用一个第三方库 : react-transition-group ;

Vue中为我们提供一些内置组件和对应的API来完成动画，利用它们我们可以方便的实现过渡动画效果；

我们来看一个案例：

  • Hello World的显示和隐藏；
  • 可以看到没有任何动画效果，十分生硬；

如果我们希望给单元素或者组件添加过渡动画，可以使用 transition 内置组件。

# 认识transition

Vue提供了 transition 的封装组件，在下列情形中，可以给任何元素和组件添加进入/离开过渡：

  • 条件渲染 (使用 v-if 或 v-show)
  • 动态组件
  • 组件根节点

使用transition修改前面案例，代码如下：

<!-- App.vue -->
<template>
  <div>
    <button @click="show = !show">toggle</button>
    <transition name="zzwi">
      <h2 v-if="show">Hello World</h2>
    </transition>
    <template v-if="show">
      <h2>Hello World</h2>
    </template>
  </div>
</template>
<script>
export default {
  data() {
    return {
      show: true
    }
  }

}
</script>
<style>
  .zzwi-enter-from,
  .zzwi-leave-to{
    opacity: 0;
  }

  /*元素默认opacity为1，可省略*/
  .zzwi-enter-to,
  .zzwi-leave-from{
    opacity: 1;
  }

  .zzwi-enter-active,
  .zzwi-leave-active{
    transition: opacity 1s ease;
  }
</style>

运行效果如下：

可以看到，transition组件自动给h2元素添加了动画，这是什么原因呢？

当 插入或删除 包含在 transition 组件中的元素时，Vue将会做以下处理：

  1. 自动嗅探目标元素是否应用了CSS过渡或者动画 ，如果有，那么在恰当的时机添加/删除CSS类名；
  2. 如果 transition 组件提供了 JavaScript钩子函数 ，这些钩子函数将在恰当的时机被调用；
  3. 如果没有找到JavaScript钩子并且也没有检测到CSS过渡/动画，DOM插入、删除操作将会 立即执行 ，不会添加动画；

# 过渡动画class

在上面 transition 包裹h2元素给其添加动画的代码中，可以发现css代码里多出了很多 "不是我们定义的class名称" ，

事实上，Vue就是帮助我们 在这些class之间来回切换完成的动画 ，

如果没有在 transition组件内 自定义 name属性 的值，这些class名称默认都是以 v- 开头，

如果像上面代码中给name赋值为zzwi，这些class名称便都以 zzwi- 开头，

这些class的默认名称和作用如下：

  • v-enter-from : 定义进入过渡的开始状态。在元素被插入之前生效，在元素被插入之后的下一帧移除。
  • v-enter-active : 定义进入过渡生效时的状态。在整个进入过渡的阶段中应用，在元素被插入之前生效，在在过渡/动画完成之后移除。这个类可以被用来定义进入过渡的进程时间，延迟和曲线函数。
  • v-enter-to : 定义进入过渡的结束状态。在元素被插入之后下一帧生效(与此同时 v-enter-from 被移除)，在过渡/动画完成之后移除。
  • v-leave-from : 定义离开过渡的开始状态。在离开过渡被触发时立刻生效，下一帧被移除。
  • v-leave-active : 定义离开过渡生效时的状态。在整个离开过渡的阶段中应用，在离开过渡被触发时立刻生效，在过渡/动画完成之后移除。这个类可以被用来定义离开过渡的过程时间，延迟和曲线函数。
  • v-leave-to : 离开过渡的结束状态。在离开过渡被触发之后下一帧生效(与此同时 v-leave-from 被删除)，在过渡/动画完成之后移除。

个人理解：

  • 在插入元素时[(v-if / v-show) 为true]，切换带 enter 的class；
  • 在移除元素时[(v-if / v-show) 为false]，切换带 leave 的class；

下图是 vue3中文文档 (opens new window) 对过渡class的图解：

前面 zzwi-enter-active 和 zzwi-leave-active 这两个class都是通过 transition 来实现的动画效果，同样的效果还可以通过 animation 来实现。

如下CSS代码所示：

.zzwi-enter-active{
  animation: bounce 1s ease;
}

.zzwi-leave-active{
  animation: bounce 1s ease reverse;
}

@keyframes bounce{
  0% {
    opacity: 0;
  }
  100% {
    opacity: 1;
  }
}

同时设置transition和animation：

  • Vue为了知道过渡的完成，内部会对 transitionend 或 animationend 进行监听，到底使用哪一个取决于元素应用的CSS规则：

  • 如果我们只是使用了其中的一个，那么Vue能自动识别类型并设置监听；

  • 但是如果我们同时使用了过渡和动画呢？

    • 并且在这个情况下可能某一个动画执行结束时，另外一个动画还没有结束；
    • 在这种情况下，我们可以设置 type 属性为animation或者transition来明确地告知Vue需要监听的类型；
    <template>
  <transition name="zzwi" 
              type="transition">
    <h2 v-if="show">Hello World</h2>
  </transition>
</template>

指定动画时间：

  • 我们可以通过 duration 属性来显示的指定过渡的时间；
  • duration可以设置两种类型的值：
    • number类型 : 同时设置进入和离开的过渡时间；
    • object类型 : 分别设置进入和离开的过渡时间；
<template>
  <transition name="zzwi" 
              type="transition"
              :duration="1000">
    <h2 v-if="show">Hello World</h2>
  </transition>
</template>

# 过渡模式mode

在transition内置组件中实现元素切换：

<template>
  <transition name="zzwi" 
              type="transition"
              :duration="{enter: 800,leave: 1000}">
    <h2 v-if="show">Hello World</h2>
    <h2 v-else>Hey zzwi~</h2>
  </transition>
</template>

运行之后，

我们来看看当前的动画在两个元素之间切换的时候存在的问题：

我们发现 Hello World 和 Hey zzwi~ 会有一段时间同时发生变化：

  • 这是因为默认情况下 进入和离开的动画 是同时发生的；

  • 如果我们刚好想要这么一个效果，那这种情况下是没有问题的；

  • 但是，如果我们 不希望同时执行 进入和离开的动画，那么我们需要设置transition的 过渡模式 ：

    • in-out : 先执行插入新元素的进入动画，再执行移除旧元素的离开动画；
    • out-in : 先执行移除旧元素的离开动画，再执行插入新元素的进入动画；

  • 示例一：in-out

<template>
  <transition name="zzwi" 
              type="transition"
              :duration="800"
              mode="in-out">
    <h2 v-if="show">Hello World</h2>
    <h2 v-else>Hey zzwi~</h2>
  </transition>
</template>
