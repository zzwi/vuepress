# Vue3组件化开发(part2)

# 非父子组件的通信

在开发中，我们构建了组件树之后，除了 父子组件之间的通信 之外，还会有 非父子组件之间 的通信。

这里介绍主要两种方式：

  • Provide/Inject ;
  • Mitt全局事件总线 ;

# 认识Provide和Inject

Provide/Inject 用于爷孙组件之间共享数据：

  • 比如有 一些深度嵌套的组件，子组件想要获取十八代(夸张)以前祖宗组件的部分内容 ；
  • 在这种情况下，如果我们仍然 将props沿着组件链逐级传递 下去，就会非常的麻烦；
  • 对应这种情况，我们可以使用 Provide和Inject ：
    • 无论层级结构有多深，父组件都可以作为其所有子组件的 依赖提供者 ；
    • 父组件有一个 provide 选项来提供数据；
    • 子组件有一个 inject 选项来开始使用这些数据；
  • 实际上，你可以将依赖注入看作是 "long range props" , 除了：
    • 父组件不需要知道哪些子组件使用它 provide 的 property ；
    • 子组件不需要知道 inject 的 property 来自哪里；

# Provide和Inject使用

案例如下，有这样一个结构：

st=>start: App.vue
op=>operation: Home.vue
ed=>end: HomeContent.vue
st(right)->op(right)->ed

# Provide响应式处理

我们先来验证一个结果：如果我们修改了上文App.vue中provide函数内this.titles的内容，即修改了data里的titles数据，那么使用这份数据的子组件HomeContent会不会响应呢？

我们会发现对应的子组件是没有进行响应的：

  • 这是因为当我们修改了titles之后，之前在provide中 引入的this.titles.length本身并不是响应式 的；

那么怎么样可以让我们的数据变成响应式的呢？

  • 可以使用响应式的一些API来完成这个功能，比如说 computed函数 ；
  • computed是vue3新特性；
provide() {
  return {
    name: "zzwi",
    title: "provide|inject",
	// 响应式
    length: computed(() => this.titles.length)
  }
},

# 全局事件总线mitt库

Vue3从实例中移除了$on、$off和$once方法，所以我们如果希望继续使用全局事件总线，要通过第三方的库：

  • Vue3官方有推荐一些库，例如 mitt 和 tiny-emitter ；
  • 接下来只简单介绍mitt；

下载安装：

npm install mitt

在根目录创建一个 utils文件夹 ，在其中封装一个工具文件 eventbus.js ：

// eventbus.js
import mitt from 'mitt'

const emitter = mitt()

export default emitter

# 插槽Slot

在开发中，我们会经常封装一个个可复用的组件：

  • 前面我们通过 组件之间的通信 来传递数据让组件进行展示；
  • 但是为了让组件具备 更强的通用性 ，我们希望不仅可以 自定义需要展示的数据 ，还可以 自定义需要展示的HTML标签 ，如div、span、button、image等；
  • 比如某些情况下我们希望组件展示的是 一个按钮button ，另外一些情况下我们希望组件展示的是 一张图片image ；
  • 总的来说，我们希望可以让使用者来决定 组件的某一块区域存放什么内容和标签元素 。

假如我们定制一个通用的导航组件 - NavBar ：

  • 这个组件分成三块区域： 左边-中间-右边 ，每块区域的内容是不固定的；
  • 左边区域可能显示一个菜单图标，可能显示一个返回<按钮，也可能什么都不显示；
  • 中间区域可能显示一个搜索框，可能显示一个列表，也可能显示一个标题……；
  • 右边可能是一个文字，可能是一个图标，也可能什么都不显示；

这个时候，就可以使用 插槽slot 来给导航组件预留3个位置，灵活显示标签元素和数据。

# 使用插槽

插槽slot：

  • 插槽的使用过程其实是 抽取共性、预留不同 ；

  • 我们会将 共同的元素、内容 依然在组件内进行封装；

  • 同时会将 不同的元素 使用slot作为占位，让外部决定到底显示什么样的元素；

  • Vue中将 <slot> 元素作为承载分发内容的出口；

  • 在封装组件中，使用特殊的元素 <slot> 就可以为封装组件开启一个插槽；

  • 该插槽插入什么内容取决于父组件如何使用；

创建一个组件 MySlotCpn.vue : 该组件中有一个插槽，我们可以在插槽中放入需要显示的内容；

我们在App.vue中使用它们：可以给slot插入普通的内容、HTML元素、其他组件，都是可以的；

简单测试，代码如下：

# 具名插槽

上面提到，当插件内有多个 <slot> 时，会把使用者插入的所有内容复制多份并分别插入每个 <slot> 中，

但是，如果我们想要达到的不是这个效果，而是插入三份内容，分别对应三个 <slot> 的那种效果，我们应该如何做呢？

这个时候就需要使用到 具名插槽 了：

  • 具名插槽顾名思义就是给插槽起一个名字， <slot> 元素有一个特殊的attribute(属性) --- name ;
  • 一个不带name的slot，会带有隐含的名字 --- default ;

用法：

  • 在 <slot> 元素上使用属性name，例如给三个 <slot> 元素分别添加 name="left" name="center" name="right"

  • 然后，当使用者插入内容时，使用 <template> 标签包裹内容，并且给标签内加上 v-slot:left 或 v-slot:center 或 v-slot:right ，指定插入内容所要替换的具名插槽

  • v-slot 只能写在 <template> 标签 或 组件的标签 上，不能写在其他HTML标签上，否则，程序会报错

[plugin:vite:vue] v-slot can only be used on components or <template> tags.
1

  • 如果使用者插入内容时没有加上 v-slot 指定具名插槽，且此时插件内所有 <slot> 都使用了name属性成为具名插槽，那么插入内容无效，

  • 只有当插件内有 <slot> 未成为具名插槽时，没有加上 v-slot 的插入内容才会在各个未命名的 <slot> 内生效

# 动态插槽名

什么是动态插槽名呢？

目前我们使用的插槽名称都是固定的，比如 v-slot:left 、 v-slot:center 等，我们可以通过 v-slot:[dynamicSlotName] 方式动态绑定一个名称；

动态绑定的示例如下：

<!-- App.vue -->
<template>
  <div>
    <my-slot-cpn>
      <!-- 动态绑定插槽名称 -->
      <template v-slot:[name]>
        普通文本
      </template>
      ...
    </my-slot-cpn>
  </div>
</template>
<script>
import MySlotCpn from './MySlotCpn.vue'
import MyElement from './MyElement.vue'
export default {
  components: {
    MySlotCpn,
    MyElement
  },
  data() {
    return {
      name: "left"
    }
  }
}
</script>

注意：具名插槽的 v-slot 跟 v-on 、 v-bind 一样，也有自己的缩写形式，即把参数name之前的所有内容 (v-slot:)替换为字符# 。

动态插槽名称的另一种用法：

  • 父组件将动态名称[name]通过props传给子组件，
  • 子组件用接收到的[name]为插槽命名，
  • 这样一来，父组件便可以动态决定[name]的值，也可以直接给插入内容指定具名插槽而无需去翻看子组件的 v-slot

代码示例如下：

<!-- App.vue 父组件-->
<template>
  <div>
    <my-slot-cpn :name="name">
      <template v-slot:left>
        普通文本
      </template>
      <template v-slot:center>
        其他内容
      </template>
      <template v-slot:right>
        <button>
          一个按钮
        </button>
      </template>
      <!-- 动态绑定具名插槽 -->
      <template v-slot:[name]>
        zzwi
      </template>
    </my-slot-cpn>
  </div>
</template>
<script>
import MySlotCpn from './MySlotCpn.vue'
export default {
  components: {
    MySlotCpn
  },
  data() {
    return {
      name: "zzwi"
    }
  }
}
</script>

# 渲染作用域

在Vue中有渲染作用域的概念：

  • 父级模板里的所有内容都是 在父级作用域中编译 的；
  • 子模版里的所有内容都是 在子作用域中编译 的；

如何理解这句话呢？

  • 在父组件的 <template> 模板中，子组件只能访问父级作用域中的内容，而无法访问自己作用域中的数据；
  • 在子组件的 <template> 模板中，子组件只能访问自己作用域中的内容；

如果我们希望 父级模板中的插槽 <slot> 可以访问到子组件中的内容 ，

Vue给我们提供了 作用域插槽 。

使用：

  • 通过 v-slot:default 的方式使用子组件中的内容

案例如下：

  1. 在App.vue中定义一个名为names的数组；
  2. 通过props传递给子组件MySlotCpn；
  3. 在MySlotCpn组件中遍历names数组；
  4. 定义插槽的prop
  5. 通过 v-slot:default 的方式获取到slot的props
  6. 使用slotProps(自定义命名)中的item和index

代码如下(使用默认插槽default)：

<!-- MySlotCpn.vue 有插槽的子组件-->
<template>
  <div class="cpn">
    <!-- 3. 在MySlotCpn组件中遍历names数组； -->
    <template v-for="(item, index) in names" :key="item">
      <!-- 4. 定义插槽的prop -->
      <slot :item="item" :index="index"></slot>
    </template>
  </div>
</template>
<script>
export default {
  props: {
    // 2. 通过props传递给子组件MySlotCpn；
    names: {
      type: Array,
      default: () => []
    }
  }
}
</script>
<style scoped>
  .cpn{
    display: flex;
    flex-direction: column;
    align-items: center;
  }
</style>

独占默认插槽的缩写

如果我们的插槽是默认插槽default，那么在使用的时候 v-slot:default="slotProps" 可以简写为 v-slot="slotProps"

<my-slot-cpn :names="names">
  <template v-slot="slotProps">
    <div>{{slotProps.index}} - {{slotProps.item}}</div>
  </template>
</my-slot-cpn>

并且如果我们的插槽只有默认插槽时，组件的标签可以被当作插槽的模板来使用，这样，我们就 可以将 v-slot 直接用在组件上 ：

<my-slot-cpn :names="names" v-slot="slotProps">
  <div>{{slotProps.index}} - {{slotProps.item}}</div>

但是，如果我们同时有默认插槽和具名插槽，那么按照完整的template来编写，不能使用上述缩写，

只要出现多个插槽，请始终为所有的插槽使用完整的基于 <template> 的语法。
