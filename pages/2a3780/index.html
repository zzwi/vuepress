# 30.手写节流函数

# 认识节流throttle函数

什么是JavaScript的节流(throttle)函数？

我的理解：

针对被频繁调用的函数，不管其被调用了多少次，在固定时间段内只执行一次，其余的都无视，简单的说，就是单位时间内，只执行一次，避免重复且频繁的调用执行影响系统整体的性能。

或者还能这样理解：我们可以把 节流 看作游戏技能的冷却时间，在这期间，不管你怎么狂按键盘或鼠标，技能都是不会发出的，只有过了冷却时间，你才能使用技能~

下面让我们手写一个自己的throttle函数吧~

先讲一下思路：

首先，我们需要设置一个间隔时间 interval ，也就是 技能冷却时间 ，然后，我们需要知道上一次函数成功执行的时间点 lastTime ，>= lastTime + interval 就是下一次函数可以成功执行的时间 canTime ，即 技能可使用状态 ，

此外，我们需要知道当前事件触发的时间点 nowTime ，这样就可以进行条件判断：

  • 当 nowTime 处于 canTime 范围时，就可以执行该触发事件对应的函数~
  • 否则，无视该事件。

思路大概理清了之后，开始写代码：

# 手写throttle版本1：

代码如下：

// 06_throttle-v1-基本实现.js
/**
 * 
 * @param {function} fn 需要节流的函数
 * @param {number} interval 毫秒，节流间隔值
 * @returns 返回真正执行的函数
 */
function throttle(fn, interval) {
  // 1. 记录上次的函数执行时间
  let lastTime = 0
  // 2. 事件触发时，执行的函数
  function _throttle() {

    // 2.1. 获取当前事件触发的时间
    const nowTime = new Date().getTime()

    /* 2.2. 使用设置的时间间隔interval
    当前事件触发的时间nowTime
    上一次函数执行时间lastTime
    计算出下一次能成功执行函数的时间点remainTime
    */ 
    const remainTime = interval - (nowTime - lastTime)
    if(remainTime <= 0) {
      // 2.3. 需要执行节流操作的函数
      fn()
      
      // 2.4. 保留这次函数执行的时间
      lastTime = nowTime
    }
  }

  return _throttle
}

引用了我们手写的throttle版本1，并将间隔时间调为2s，效果如下：

可以看到，我们一开始拖动进度条，就执行了一次函数，在拖动过程中，2s时间第一次过去也执行了一次函数，最后在距上一次函数执行不到2s时拖动结束，版本1并没有执行最后一次函数。

为什么事件第一次触发对应的函数能立即执行呢？

因为 第一次通过 new Date().getTime() 获取到的 nowTime 数值非常大，导致第一次计算 remainTime 的值小于0，条件判断为true，函数立即执行。

之后的时间在第一次 nowTime 的基础上正常累加，所以后续的节流函数正常工作。

# 手写throttle版本2：

显然，版本1并不完善，我们需要为它增添一些功能，先从控制第一次事件触发是否执行函数开始~

通过传递参数 leading 来控制第一次事件触发时是否立即执行对应函数。

因为后续还会通过传递参数 trailing 来控制最后一次函数执行，所以这里就将这两个参数都打包放进对象 options 里一起传递了。

代码如下：

// 07_throttle-v2-leading实现.js
/**
 * 
 * @param {function} fn 需要节流的函数
 * @param {number} interval 毫秒，节流间隔值
 * @param {object} options 包含传递参数的对象
 * @returns 返回真正执行的函数
 */
function throttle(fn, interval, options = { leading: true, trailing: false }) {
  let lastTime = 0
  // 解构通过options传递的参数
  const { leading, trailing } = options

  function _throttle() {
    const nowTime = new Date().getTime()
    // 事件第一次触发时对应的函数是否执行
    if(!lastTime && !leading) lastTime = nowTime

    const remainTime = interval - (nowTime - lastTime)
    if(remainTime <= 0) {
      fn()
      lastTime = nowTime
    }
  }

  return _throttle
}

# 手写throttle版本3：

接下来，我们再继续为throttle函数添加 控制最后一次函数是否执行 的功能，通过传递 trailing 参数进行控制，前面提过，将 trailing 和 leading 一起写进了 options 对象中。

使用 setTimeout 来实现 trailing 的控制，传入 interval 作为延时执行的时间，让最后一次事件触发的对应函数(冷却时间未结束)能在 interval 过后(冷却时间结束) 执行 一次。

另外，一旦函数成功执行，就去除宏任务中多余的 setTimeout ，以免触发一次的函数执行多次。

代码如下：

// 08_throttle-v3-trailing实现.js
/**
 * 
 * @param {function} fn 需要节流的函数
 * @param {number} interval 毫秒，节流间隔值
 * @param {object} options 包含传递参数的对象
 * @returns 返回真正执行的函数
 */
function throttle(fn, interval, options = { leading: true, trailing: false }) {

  let lastTime = 0

  const { leading, trailing } = options
  /* 创建了一个名为timer的容器，
  用来存放上一次的定时器，
  同时是闭包的自由变量*/
  let timer = null

  function _throttle() {

    const nowTime = new Date().getTime()
	// 事件第一次触发时对应的函数是否立即执行
    if(!lastTime && !leading) lastTime = nowTime

    const remainTime = interval - (nowTime - lastTime)
    if(remainTime <= 0) {
      // 此时对应函数被成功执行，取消之前的定时器
      if(timer) {
        clearTimeout(timer)
        /* 需要将timer置为null，
        这样下次才可以给容器timer设置新的定时器*/
        timer = null
      }

      fn()

      lastTime = nowTime

      // 不再执行后面的判断语句
      return
    }

    // 设置定时器，执行未满间隔时间的最后一次函数
    // !timer : 只加一个定时器
    if(trailing && !timer) {
      timer = setTimeout(() => {
        // 将timer设置为null，下次才能开启一个新的定时器
        timer = null
        lastTime = !leading ? 0 : new Date().getTime()
        fn()
      }, remainTime)
    }
  }

  return _throttle
}

# 手写throttle版本4：

我们还能继续完善，现在我们需要让throttle节流函数可以接收 需要被节流的函数 的 event参数 ，以及修改 需要被节流的函数 的this指向。

可以通过apply(或call)方法实现。

关于版本4，我打算拿搜索框案例来讲，

代码如下:

// 09_throttle-v4-this-参数.js
/**
 * 
 * @param {function} fn 需要节流的函数
 * @param {number} interval 毫秒，节流间隔值
 * @param {object} options 包含传递参数的对象
 * @returns 返回真正执行的函数
 */
function throttle(fn, interval, options = { leading: true, trailing: false }) {
  let lastTime = 0
  const { leading, trailing } = options
  let timer = null

  // 传入参数
  function _throttle(...args) {

    const nowTime = new Date().getTime()
    if(!lastTime && !leading) lastTime = nowTime

    const remainTime = interval - (nowTime - lastTime)
    if(remainTime <= 0) {
      if(timer) {
        clearTimeout(timer)
        timer = null
      }

      // 修改this指向
      // 传入参数
      fn.apply(this, args)
      
      lastTime = nowTime

      return
    }

    if(trailing && !timer) {
      timer = setTimeout(() => {
        timer = null
        lastTime = !leading ? 0 : new Date().getTime()
        // 修改this指向
        // 传入参数
        fn.apply(this, args)
      }, remainTime)
    }
  }

  return _throttle
}

# 手写throttle版本5：

让我们继续~

我们还可以给throttle函数添加一个取消功能，用来取消还未执行的 setTimeout ，即 取消 参数 trailing 为true时还未执行的最后一次函数(未满interval间隔时间)，

可以将取消功能添加为 _throttle函数的方法。

代码如下：

// 10_throttle-v5-取消功能.js
/**
 * 
 * @param {function} fn 需要节流的函数
 * @param {number} interval 毫秒，节流间隔值
 * @param {object} options 包含传递参数的对象
 * @returns 返回真正执行的函数
 */
 function throttle(fn, interval, options = { leading: true, trailing: false }) {
  // 1. 记录上次的函数执行时间
  let lastTime = 0
  // 解构通过options传递的参数
  const { leading, trailing } = options
  /* 创建了一个名为timer的容器，
  用来存放上一次的定时器，
  同时是闭包的自由变量*/
  let timer = null
  // 2. 事件触发时，执行的函数
  function _throttle(...args) {

    // 2.1. 获取当前事件触发的时间
    const nowTime = new Date().getTime()
    // 事件第一次触发时对应的函数是否执行
    if(!lastTime && !leading) lastTime = nowTime

    /* 2.2. 使用设置的时间间隔interval
    当前事件触发的时间nowTime
    上一次函数执行时间lastTime
    计算出下一次能成功执行函数的时间点remainTime
    */ 
    const remainTime = interval - (nowTime - lastTime)
    if(remainTime <= 0) {
      // 此时对应函数被节流执行，取消之前的定时器
      if(timer) {
        clearTimeout(timer)
        timer = null
      }
      // 2.3. 需要执行节流操作的函数
      // 修改this指向
      // 传入参数
      fn.apply(this, args)
      
      // 2.4. 保留这次函数执行的时间
      lastTime = nowTime

      // 不再执行后面的判断语句
      return
    }

    // 设置定时器，执行未满间隔时间的最后一次函数
    
    if(trailing && !timer) {
      timer = setTimeout(() => {
        timer = null
        lastTime = !leading ? 0 : new Date().getTime()
        fn.apply(this, args)
      }, remainTime)
    }
  }

  // 添加取消功能
  _throttle.cancel = function() {
    if(timer) clearTimeout(timer)
    timer = null
    lastTime = 0
  }

  return _throttle
}

# 手写throttle版本6：

最后再给throttle函数添加函数返回值，打印一下搜索框input监听事件对应的执行函数它的返回值。

这里有两种方案：

  • 方案一是传入参数resultCallback
  • 方案二是使用Promise

// 11_throttle-v6-函数返回值.js
/**
 * 
 * @param {function} fn 需要节流的函数
 * @param {number} interval 毫秒，节流间隔值
 * @param {object} options 包含传递参数的对象
 * @param {function} resultCallback 获取需要节流的函数它的返回值
 * @returns 返回真正执行的函数
 */
 function throttle(fn, interval, options = { leading: true, trailing: false }, resultCallback) {
  // 1. 记录上次的函数执行时间
  let lastTime = 0
  // 解构通过options传递的参数
  const { leading, trailing } = options
  /* 创建了一个名为timer的容器，
  用来存放上一次的定时器，
  同时是闭包的自由变量*/
  let timer = null
  // 2. 事件触发时，执行的函数
  function _throttle(...args) {

    // 2.1. 获取当前事件触发的时间
    const nowTime = new Date().getTime()
    // 事件第一次触发时对应的函数是否执行
    if(!lastTime && !leading) lastTime = nowTime

    /* 2.2. 使用设置的时间间隔interval
    当前事件触发的时间nowTime
    上一次函数执行时间lastTime
    计算出下一次能成功执行函数的时间点remainTime
    */ 
    const remainTime = interval - (nowTime - lastTime)
    if(remainTime <= 0) {
      // 此时对应函数被节流执行，取消之前的定时器
      if(timer) {
        clearTimeout(timer)
        timer = null
      }
      // 2.3. 需要执行节流操作的函数
      // 修改this指向
      // 传入参数
      // 接收fn的返回值
      const result = fn.apply(this, args)
      if(resultCallback) resultCallback(result)
      // 2.4. 保留这次函数执行的时间
      lastTime = nowTime

      // 不再执行后面的判断语句
      return
    }

    // 设置定时器，执行未满间隔时间的最后一次函数
    if(trailing && !timer) {
      timer = setTimeout(() => {
        timer = null
        lastTime = !leading ? 0 : new Date().getTime()
        // 接收fn的返回值
        const result = fn.apply(this, args)
        if(resultCallback) resultCallback(result)
      }, remainTime)
    }
  }

  // 添加取消功能
  _throttle.cancel = function() {
    if(timer) clearTimeout(timer)
    timer = null
    lastTime = 0
  }

  return _throttle
}

 * 
 * @param {function} fn 需要节流的函数

  // 2. 事件触发时，执行的函数
  function _throttle(...args) {
    // 返回一个Promise
    return new Promise((resolve, reject) => {
      // 2.1. 获取当前事件触发的时间
      const nowTime = new Date().getTime()
      // 事件第一次触发时对应的函数是否立即执行

      /* 2.2. 使用设置的时间间隔interval
      当前事件触发的时间nowTime

          clearTimeout(timer)

        // 修改this指向
        // 传入参数
        // 接收fn的返回值
        const result = fn.apply(this, args)
        // promise接收返回值
        resolve(result)
        // 2.4. 保留这次函数执行的时间
        lastTime = nowTime

        // 不再执行后面的判断语句
        return
      }

      // 设置定时器，执行未满间隔时间的最后一次函数
      if(trailing && !timer) {
        timer = setTimeout(() => {
          timer = null
          lastTime = !leading ? 0 : new Date().getTime()
          // 接收fn的返回值
          const result = fn.apply(this, args)
          // promise接收返回值
          resolve(result)
        }, remainTime)
      }
    })

  }

  // 添加取消功能
  _throttle.cancel = function() {
    if(timer) clearTimeout(timer)
    timer = null
    lastTime = 0
  }

  return _throttle
}
