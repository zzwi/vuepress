JSON-数据存储

# 26.JSON-数据存储

# JSON的由来

在目前的开发中，JSON是一种非常重要的 数据格式 ，它并不是编程语言，而是一种可以在服务器和客户端之间传输的数据格式。

JSON的全称是 JavaScript Object Notation( JavaScript对象符号 )：

  • JSON是有Douglas Crockford构想和设计的一种轻量级资料交换格式，算是JavaScript的一个子集
  • 虽然JSON被提出来的时候主要应用于JavaScript中，但是目前已经独立于编程语言，可以在各个编程语言中使用
  • 很多编程语言都实现了将JSON转成对应模型的方式

其他的传输格式：

  • XML：在早期的网络传输中主要是使用XML来进行数据交换的，但是这种格式在解析、传输等各方面都弱于JSON，所以目前已经很少被使用了
  • Protobuf：另外一个在网络传输中目前已经越来越多被使用的传输格式是protobuf，但是直到2021年的3.x版本才支持JavaScript，所以目前在前端使用的较少

目前JSON被使用的场景也越来越多：

  • 网络数据的传输JSON数据
  • 项目的某些配置文件
  • 非关系型数据库(NoSQL)将json作为存储格式
  • 小程序的app.json

# JSON的序列化

某些情况下我们希望将JavaScript中的复杂类型转化成JSON格式的字符串，这样方便对其进行处理：

  • 比如我们希望将一个对象保存到localStorage中
  • 但是如果我们直接存放一个对象，这个对象会被转化成[object Object]形式的字符串，并不是我们想要的结果
// JavaScript(浏览器环境)
const obj = {
  name: "zzwi",
  age: 18,
  target: {
    name: "excellent"
  },
  hobbies: ["读书","跑步"]
}

localStorage.setItem("obj",obj)

const getObj = localStorage.getItem("obj")
console.log(getObj) // [object Object]

# stringify方法

JSON.stringify()方法将一个JavaScript对象或值转换为JSON字符串。

stringify方法的第一个参数是需要被转为字符串格式的数据。

stringify方法的第二个参数replacer：

  • 如果指定的replacer是数组，则可选择性地仅包含数组指定的属性
// JavaScript(浏览器环境)
const obj = {
  name: "zzwi",
  age: 18,
  target: {
    name: "excellent"
  },
  hobbies: ["读书","跑步"]
}

// 转成JSON字符串
const objString = JSON.stringify(obj,["name","age"])
console.log(objString)
// 字符串
// {"name":"zzwi","age":18}

# toJSON方法

如果对象本身包含toJSON方法，那么会直接使用toJSON方法的结果

// JavaScript(浏览器环境)
const obj = {
  name: "zzwi",
  age: 18,
  target: {
    name: "excellent"
  },
  hobbies: ["读书","跑步"],
  // toJSON方法
  toJSON: function() {
    return 123456
  }
}

const objString = JSON.stringify(obj)
console.log(objString) // 123456

const objString1 = JSON.stringify(obj,["name","age"])
console.log(objString1) // 123456

const objString2 = JSON.stringify(obj,(key,value) => {
  if(key === "age") {
    return 22
  }
  return value
})
console.log(objString2) // 123456

const objString3 = JSON.stringify(obj,null,2)
console.log(objString3) // 123456

# parse方法

JSON.parse()方法用来解析JSON字符串，构造由字符串描述的JavaScript值或对象

parse方法的第一个参数是需要被解析的JSON字符串

parse方法的第二个参数reviver：

  • reviver是可选的
  • 如果选定的reviver参数是一个函数，则可以用于在返回之前对所得到的解析值执行 变换 (操作)。
// JavaScript(浏览器环境)
const obj = {
  name: "zzwi",
  age: 18,
  target: {
    name: "excellent"
  },
  hobbies: ["读书","跑步"]
}

// 转为JSON字符串
const objString = JSON.stringify(obj)
// 解析JSON字符串
const info = JSON.parse(objString,(key,value) => {
  if(key === "name") {
    return "bill"
  }
  return value
})
console.log(info)
// 对象
// {name: 'bill', age: 18, target: {…}, hobbies: Array(2)}

# 利用JSON序列化深拷贝

我们使用JSON.stringify()转化为字符串再使用JSON.parse()解析回来的新对象，和之前的对象并不是同一个对象

相当于是进行了一次深拷贝

什么是 深拷贝 ？是否有 浅拷贝 呢？

我们知道，每个对象在堆内存中都有一块对应的内存空间，而标识符里存储的只是指向该空间的地址，

平时我们使用的引用赋值：

const obj = {name:"zzwi"}
const foo = obj
1
2

只是单纯地将地址赋值给了另外一个标识符(变量名称)

它们两者(obj和foo)实际表示的还是同一个对象。

这里先回答一下第二个疑问，是否有 浅拷贝 呢？

当然有。

// JavaScript(node环境)
const obj = {
  name: "zzwi",
  age: 18,
  foo: {
    name: "zed"
  }
}
// 浅拷贝
const obj2 = {...obj}

// 基本类型属性
obj.name = "bill"
console.log(obj2.name) // zzwi

// 引用类型属性
obj.foo.name = "zzwi00"
console.log(obj2.foo.name) // zzwi00

知晓了浅拷贝，那么你现在能猜到深拷贝了吗？

没错，就是连同包含的引用类型属性也拥有一个独立的内存空间。

// JavaScript(node环境)
const obj = {
  name: "zzwi",
  age: 18,
  foo: {
    name: "zed"
  }
}

// 利用JSON序列化深拷贝
const objString = JSON.stringify(obj)
const info = JSON.parse(objString)

// 基本数据类型
obj.name = "bill"
console.log(info.name) // zzwi

// 引用类型属性
obj.foo.name = "zzwi00"
console.log(info.foo.name) // zed
