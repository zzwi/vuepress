方式一： v-if 判断

<!-- App.vue 父组件-->
<template>
  <div>
    <button v-for="item in tabs"
            :key="item"
            @click="tabClick(item)"
            :class="{active: currentTab === item}">
            {{item}}        
    </button>

    <!-- v-if判断 -->
    <template v-if="currentTab === 'home'">
      <home></home>
    </template>
    <template v-else-if="currentTab === 'about'">
      <about></about>
    </template>
    <template v-else>
      <category></category>
    </template>
  </div>
</template>
<script>
import Home from './Home.vue'
import About from './About.vue'
import Category from './Category.vue'

export default {
  components: {
    Home,
    About,
    Category
  },
  data() {
    return {
      tabs: ["home", "about", "category"],
      currentTab: "home"
    }
  },
  methods: {
    tabClick(item) {
      this.currentTab = item
    }
  }
}
</script>

# 认识keep-alive

keep-alive是vue的 内置组件 ，可以 保存组件的状态 。

对前面的案例进行改造，在About组件中增加一个按钮counter，内有数字，点击该按钮可以递增；

当我们将counter点到10，然后切换到home再切换回about时，状态是否可以保持呢？

  • 答案是否定的；
  • 因为默认情况下，我们在切换组件后，about组件会被销毁掉，再次回来时会重新创建组件；
  • 效果：

但是，在某些情况下我们希望继续保持组件的状态，而不是销毁掉，这个时候我们就可以使用一个内置组件：keep-alive

代码：

<keep-alive>
  <component :is="currentTab"></component>
</keep-alive>
1
2
3

效果：

可以看到，counter内的数字10被保存了下来。

keep-alive属性

  • include ：string|RegExp|Array。只有名称匹配的组件会被缓存；
  • exclude ：string|RegExp|Array。任何名称匹配的组件都不会被缓存；
  • max ：number|string。最多可以缓存多少组件实例，一旦达到这个数字，那么缓存组件中最近没有被访问的实例会被销毁；

include 和 exclude prop 允许组件有条件地缓存：

  • 二者都可以用 逗号分隔字符串、正则表达式或一个数组 来表示；
  • 匹配首先检查组件自身的 name 选项；
<!-- 逗号分隔字符串(不要加空格) -->
<keep-alive include="a,b">
  ...
</keep-alive>
<!-- 正则表达式 (使用v-bind) -->
<keep-alive :include="/a|b/">
  ...
</keep-alive>
<!-- 一个数组 (使用v-bind) -->
<keep-alive :include="['a', 'b']">
  ...
</keep-alive>

# webpack的代码分包

默认的打包过程：

  • 默认情况下，在构建整个组件树的过程中，因为组件和组件之间是 通过模块化直接依赖 的，那么webpack在打包时就会将组件模块打包到一起(比如一个app.js文件)；
  • 随着项目的不断扩大，当app.js文件中的内容过大时，会造成 首屏的渲染速度变慢 ；

实际上，webpack在打包代码时，默认情况下会将代码打包成两个文件：

  • 一个是 app.一串hash值.js ，这个文件里打包的是所有我们 自己编写的逻辑代码 ；

  • 一个是 chunk-vendors.一串hash值.js ，这个文件里打包的是所有 第三方库的代码 ；

  • 当这两个文件过大时，首屏的渲染速度变慢；

所以，对于一些不需要立即使用的组件，我们可以单独对它们进行拆分，拆分成一些小的代码块 chunk.js ,

这些chunk.js会在需要时从服务器上下载下来，并且运行代码，显示相关内容。

如何手动分包？

  • 使用import函数
// 通过import函数导入的模块, 后续webpack对其进行打包的时候就会进行分包的操作
// import函数返回一个promise
import("./05_slot_dynamic_component/utils/add.js").then(res => {
  console.log(res.sum(20,30))
})

# Vue中实现异步组件

如果我们的项目过大了，对于某些组件我们希望通过 异步的方式 来进行加载(目的是可以对其进行 分包处理 )，为此，Vue中给我们提供了一个函数： defineAsyncComponent .

defineAsyncComponent接受两种类型的参数：

  • 类型一：工厂函数，该工厂函数需要返回一个Promise对象；
  • 类型二：接受一个对象类型，对异步函数进行配置；

类型一：常用写法

<!-- App.vue -->
...

import { defineAsyncComponent } from 'vue'

export default {
  components: {
    AsyncHome

# $refs的使用

某些情况下，我们在组件中想要 直接获取到元素对象或者子组件实例 ：

  • 但是，在Vue开发中我们是不推荐进行DOM操作的；
  • 这个时候，我们可以给元素或者组件绑定一个 ref 的属性(attribute);

组件实例有一个 $refs 属性：

  • 它是一个对象，持有 注册过 ref 的所有DOM元素和组件实例。
<!-- App.vue 父组件 -->
<template>
  <div>
    <h2 ref="title">ref element test</h2>
    <my-cpn ref="myCpn"></my-cpn>
    <button @click="getElement">点击获取元素</button>
  </div>
</template>
<script>
import MyCpn from './MyCpn.vue'

export default {
  components: {
    MyCpn
  },
  methods: {
    getElement() {
      // 访问元素
      console.log(this.$refs.title);
      // 访问组件实例
      console.log(this.$refs.myCpn);
      console.log(this.$refs.myCpn.message);
      this.$refs.myCpn.showMessage()
    }
  }
}
</script>

# 生命周期简述

生命周期：

  • 每个组件都可能会经历从创建、挂载、更新、卸载等一系列的过程；
  • 在这个过程中的某一个阶段，使用者可能会想要添加一些属于自己的代码逻辑(比如组件创建完成后向服务器请求数据)；
  • 但是我们如何知道组件此时在哪一阶段呢？Vue给我们提供了组件的生命周期函数；

生命周期函数：

  • 生命周期函数是一些钩子函数，在某个时间会被Vue源码内部进行回调；
  • 通过对生命周期函数的回调，我们可以知道目前组件正在经历什么阶段；
  • 我们还可以通过生命周期函数在该生命周期中编写属于自己的逻辑代码；

# 组件的v-model

我们在input等表单元素中可以使用v-model来完成双向绑定：

  • 这里v-model默认帮助我们完成了两件事，
  • v-bind:value 的数据绑定和 @input 的事件监听。

Vue支持我们在自己的组件上使用v-model，当然，需要我们给组件内添加一些特殊的逻辑代码以实现v-model。

我们在组件上写入

v-model="message"

这里v-model也同样帮我们做了两件事：

:model-value="message" @update:model-value="textChange"

现在对这两件事进行简单介绍，

  • :model-value="message" 是将数据message通过props传递给子组件的modelValue，
  • @update:model-value="textChange" 是接收子组件发送出来的事件update:model-value并触发本页面事件textChange

有两种实现方式：

方式一： :value和@input

<!-- MyInput.vue 子组件 -->
<template>
  <div>
    <input :value="modelValue" @input="inputChange">
  </div>
</template>
<script>
export default {
  name: "MyInput",
  emits: ["update:modelValue"],
  props: {
    modelValue: {
      type: String
    }
  },
  methods: {
    inputChange(event) {
      this.$emit("update:modelValue", event.target.value)
    }
  }
}
</script>

    }
  }

# 绑定多个属性

我们现在通过v-model直接绑定了一个属性，如果我们希望绑定多个属性呢？

  • 也即我们希望在一个组件上使用多个v-model；

  • 我们知道，默认情况下的v-model其实是 动态绑定了modelValue属性 并 接收监听了update:modelValue的事件 ；

  • 当我们希望绑定更多属性时，可以 给v-model传入一个参数 ，这个参数就是我们所绑定属性的名称；

  • 就像这样：

    v-model:title="title"

    这行代码做了两件事情，

    • 绑定了title属性；
    • 监听了update:title事件；

代码示例如下：

<!-- MyInput.vue 子组件 -->
<template>
  <div>
    <label for="value">message: </label>
    <input id="value" v-model="value">
    <br/>
    <label for="title">title: </label>
    <input id="title" v-model="title">    
  </div>
</template>
<script>
export default {
  name: "MyInput",
  // 注册发送事件
  emits: ["update:modelValue", "update:title"],
  props: {
    modelValue: {
      type: String
    },
    title: {
      type: String
    }
  },
  computed: {
    value: {
      get() {
        return this.modelValue
      },
      set(value) {
        this.$emit("update:modelValue", value)
      }
    },
    title: {
      get() {
        return this.title
      },
      set(value) {
        this.$emit("update:title", value)
      }
    }
  }
}
</script>

    margin-top: 20px;
  }
