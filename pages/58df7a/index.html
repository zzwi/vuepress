# 10.ES6中类的使用

# 认识class定义类

按照构造函数形式创建 类 ，不仅仅和编写普通的函数过于相似，而且代码并不容易理解。

  • 在ES6中(ECMAScript2015)新的标准中使用了class关键字来直接定义类
  • class关键字本质上是构造函数、原型链的语法糖

使用class来定义一个类

// JavaScript
// 方法1

# class的构造方法

想在使用new创建对象的时候给类传递一些参数，这时候会用到class内部的 constructor

  • 每个类都可以有一个自己的构造函数(方法)，这个方法的名称是固定的 constructor
  • 当我们通过new操作符，操作一个类的时候会调用这个类的构造函数 constructor
  • 每个类只能有一个构造函数，如果包含多个构造函数，那么会抛出异常

当我们通过new关键字操作类的时候，会调用这个constructor函数，并且执行如下操作：

  1. 在内存中创建一个新的对象（空对象）
  2. 这个对象内部的[[prototype]]属性会被赋值为该类的prototype属性
  3. 构造函数内部的this，会指向创建出来的新对象
  4. 执行构造函数的内部代码（函数体代码）
  5. 如果构造函数没有返回非空对象，则返回创建出来的新对象

代码如下：

// JavaScript
// 类的声明
class Person {
  // 类的构造方法
  // 注意: 一个类只能有一个构造函数
  // 1.在内存中创建一个对象 moni = {}
  // 2.将类的原型prototype赋值给创建出来的对象 moni.__proto__ = Person.prototype
  // 3.将对象赋值给函数的this: new绑定 this = moni
  // 4.执行函数体中的代码
  // 5.自动返回创建出来的对象
  constructor(name, age) {
    this.name = name
    this.age = age
  }
}


var p1 = new Person("why", 18)
var p2 = new Person("kobe", 30)
console.log(p1) // Person { name: 'why', age: 18 } 
console.log(p2) // Person { name: 'kobe', age: 30 }

# class中的方法定义

# 1.实例方法

在上面我们定义的属性都是直接放到了this上，new操作时这些属性会被放到创建出来的新对象中

  • 实例方法放到原型上，可以被多个实例共享
// JavaScript(node环境)
// 类的声明
class Person {
  constructor(name, age) {
    this.name = name
    this.age = age
  }
  eating() {
    console.log(this.name+" eating~");
  }
  running() {
    console.log(this.name+" running~");
  }
}

var p = new Person("bill",18)
p.eating() // bill eating~
p.running() // bill running~
console.log(p.__proto__.eating) // [Function: eating]
console.log(p.__proto__.running) // [Function: running]

# 2.类的访问器方法

对象的属性描述符可以添加[[setter]]和[[getter]]函数，类也是可以的

// JavaScript(node环境)
// 类的声明
class Person {
  constructor(name, age) {
    this.name = name
    this.age = age
    this._address = "广州市"
  }
  get address() {
    console.log("执行一次访问操作")
    return this._address  
  }
  set address(newAddress) {
    console.log("执行一次设置操作")
    this._address = newAddress
  }
}

var p = new Person("bill",18)
console.log(p.address) // [执行一次访问操作] 广州市
p.address = "普宁市"
console.log(p.address) // [执行一次设置操作 执行一次访问操作] 普宁市

# 3.类的静态方法

静态方法通常用于定义直接使用类来执行的方法(Person.×××())，不需要有类的实例，使用 static 关键字来定义

// JavaScript(node环境)
var names = ["abc","cba","nba"]
// 类的声明
class Person {
  constructor(name, age) {
    this.name = name
    this.age = age
    this._address = "广州市"
  }
  static randomPerson() {
    var nameIndex = Math.floor(Math.random()*names.length)
    var name = names[nameIndex]
    var age = Math.floor(Math.random() * 100)
    return new Person(name,age)
  }
}

for(var i = 0;i < 50;i++) {
  console.log(Person.randomPerson());// ...
}

// 类的声明
class Person {
  constructor(name, age) {
    this.name = name
    this.age = age
    this._address = "广州市"
  }

  // 1.普通的实例方法
  // 创建出来的新对象进行访问

    console.log(this.name + " eating~")

    console.log(this.name + " running~")
  }

    console.log("执行一次访问操作")
    return this._address  
  }
  set address(newAddress) {
    console.log("执行一次设置操作")
    this._address = newAddress
  }

  // 3.类的静态方法(类方法)
  static randomPerson() {
    var nameIndex = Math.floor(Math.random()*names.length)
    var name = names[nameIndex]
    var age = Math.floor(Math.random() * 100)
    return new Person(name,age)
  }
}

p.eating() // bill eating~
p.running() // bill running~

console.log(p.address) // [执行一次访问操作] 广州市
p.address = "普宁市"
console.log(p.address) // [执行一次设置操作 执行一次访问操作] 普宁市

for(var i = 0;i < 50;i++) {
  console.log(Person.randomPerson());
}

# ES6类的继承

前面我们在ES5中实现了相对满意的继承机制，但是过程非常繁琐

ES6中新增了 extends 关键字，可以帮助我们方便地实现继承

// JavaScript(node环境)
class Person {}
class Student extends Person {}
1
2
3

super 关键字

注意： 在子(派生)类的构造函数中使用this或者返回默认对象之前，必须先通过 super 调用父类的构造函数！

super 的使用位置有三个：子类的构造函数、实例方法、静态方法

代码如下：

// JavaScript(node环境)
class Person {
  // 构造函数
  constructor(name,age) {
    this.name = name
    this.age = age
  }
  // 实例方法
  running() {
    console.log(this.name+" running~");
  }
  eating() {
    console.log(this.name+" eating~");
  }
  // 静态方法
  static staticMethod() {
    console.log("PersonStaticMethod");
  }
}

class Student extends Person {
  constructor(name,age,sno) {
    // 1.super：子类的构造函数
    super(name,age)
    this.sno = sno
  }
  studying() {
    // 2.super：实例方法
    super.running()
    console.log(this.name+" studying~");
  }
  /**
   * 子类对父类方法的重写，不影响父类
   * 调用方法时优先调用子类同名方法
   */
  eating() {
    console.log("Student "+this.name+" eating~");
  }
    
  // 重写静态方法
  static staticMethod() {
    // 3.super: 静态方法
    super.staticMethod()
    console.log("StudentStaticMethod");
  }  
}

var stu = new Student("zzwi",18,111)
console.log(stu) // Student { name: 'zzwi', age: 18, sno: 111 }
stu.studying() // zzwi running~ \n zzwi studying~
stu.eating() // Student zzwi eating~
Student.staticMethod() // PersonStaticMethod \n StudentStaticMethod

# 继承内置类

也可以让我们的类继承内置类，比如：Array

// JavaScript
class ZyArray extends Array {
  firstItem() {
    return this[0]
  }
  lastItem() {
    return this[this.length-1]
  }
}

var arr = new ZyArray(0,1,2)
console.log(arr.firstItem()) // 0
console.log(arr.lastItem()) // 2

# ES6类的混入mixin

JavaScript的类只支持单继承：也就是只能有一个父类

  • 那么在开发中我们需要在一个类中添加更多相似的功能时，应该如何来做呢？
  • 这个时候我们可以使用混入(mixin)

代码如下：

// JavaScript(node环境)
// 父类
class Person {
  constructor(name,age) {
    this.name = name
    this.age = age
  }
}

// 混入running方法
function mixinRunner(BaseClass) {
  class NewClass extends BaseClass {
    running() {
      console.log(this.name+" running~");
    }
  }
  return NewClass
}

// 混入studying方法
function mixinStudyer(BaseClass) {
  return class extends BaseClass {
    studying() {
      console.log(this.name+" studying~");
    }
  }
}

// 子类
// 混入的本质：继承套娃
class Student extends mixinStudyer(mixinRunner(Person)) {
  constructor(name,age,sno) {
    super(name,age)
    this.sno = sno
  }
}

// 实例验证
var stu = new Student("zzwi",18,111)
console.log(stu) // Student { name: 'zzwi', age: 18, sno: 111 }
stu.running() // zzwi running~
stu.studying() // zzwi studying~
