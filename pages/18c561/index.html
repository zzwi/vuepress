# JavaScript函数的this指向

# this是动态绑定的

只有当函数执行时才能确定this的指向，this是在运行时被绑定的

# 对象中的方法/函数

// JavaScript
var obj = {
    name: "zzwi",
    studying: function(){
        console.log(this.name+"在学习");
    },
    running: function(){
        console.log(this.name+"在跑步");
    }
};

obj.studying();// zzwi在学习
obj.running();// zzwi在跑步

var fnS = obj.studying;
var fnR = obj.running;

# this的四种绑定规则

this的四种绑定规则分别为：

  • 默认绑定
  • 隐式绑定
  • 显式绑定
  • new绑定

# 默认绑定

默认绑定：

  • 独立函数调用

如下五段示例代码：

示例一：

// JavaScript
function foo(){
    console.log(this)
}
foo() // window

# 显式绑定

显式绑定：

  • call和apply在执行函数时，是可以明确地绑定this，这个绑定规则被称之为显式绑定

  • bind在执行函数时，可以返回一个明确绑定this的新函数

如下三段示例代码：

示例一：

// JavaScript
function foo(){
    console.log(this)
}
var obj1 = {
    name: 'obj1'
}
foo.call(obj1) // obj1
foo.apply(obj1) // obj1

# new绑定

new绑定：

  • 我们通过一个new关键字绑定一个函数(构造器)，这个时候this是在调用这个构造器时创建出来的对象
  • this === 创建出来的对象
  • 这个绑定过程就是new绑定

使用new关键字来调用函数，会执行如下的操作：

  1. 创建一个全新的对象;
  2. 这个新对象会被执行prototype连接;
  3. 这个新对象会绑定到函数调用的this上(this的绑定在这个步骤完成);
  4. 如果函数返回原始值，则将其忽略
  5. 如果函数返回对象，便正常处理

示例代码：

// JavaScript
function Person(name,age){
    this.name = name
    this.age = age
}

var p1 = new Person('zzwi',18)
var p2 = new Person('bill',28)
console.log(p1.name,p1.age);// zzwi 18
console.log(p2.name,p2.age);// bill 28

var obj = {
    name: 'zzwi',
    foo: function(){
        console.log(this);
    }
}
new obj.foo() // foo

手写new操作符：

// JavaScript
function mynew(Func, ...args) {
    // 1.创建一个新对象
    const obj = {}
    // 2.新对象原型指向构造函数原型对象
    obj.__proto__ = Func.prototype
    // 3.将构建函数的this指向新对象
    let result = Func.apply(obj, args)
    // 4.根据返回值判断
    return result instanceof Object ? result : obj
}
